# 1. 문제인식 (Problem)

## 1-1. 창업아이템 배경 및 필요성

### (1) 급성장하는 피트니스 시장과 해결되지 않은 구조적 문제

글로벌 피트니스 시장은 2025년 약 **2,570억 달러(약 360조 원)** 규모이며, 글로벌 헬스·피트니스 클럽 시장은 **1,256억 달러**, CAGR 9.2%로 성장 중이다. 특히 AI 피트니스 시장은 2024년 **98억 달러** → 2034년 **461억 달러** (CAGR 16.8%, InsightAce Analytic)로 빠르게 성장하고 있으며, 디지털 헬스 투자에서 AI가 차지하는 비중은 2025년 기준 **54%**에 달한다. 국내 피트니스 시장 역시 **47.5억 달러**, CAGR 9.96% (Mordor Intelligence), 스마트 피트니스 CAGR은 **19.6%**로 더욱 가파르게 성장 중이다.

그러나 이 거대한 시장의 이면에는 해결되지 않은 핵심 문제들이 존재한다.

**[문제 1] 운동 데이터의 수동 기록 — "기억에 의존하는 운동"**

헬스장에서 웨이트 트레이닝을 수행하는 대다수의 이용자는 운동 기록을 체계적으로 관리하지 못한다. 세트마다 폰을 꺼내 숫자를 입력하는 것은 운동 흐름을 깨뜨리며, 대부분 2주 만에 포기한다. "지난주에 벤치프레스를 몇 kg으로 했는지" 기억하지 못한 채 매번 감(感)에 의존하여 운동을 반복하는 것이 현실이다.

이는 **과학적 점진적 과부하(Progressive Overload)의 실패**로 직결된다. 기록 없이는 중량이나 횟수를 점진적으로 늘려나갈 근거가 없으며, 정체기 → 회의감 → 운동 중단의 악순환을 유발한다.

피카는 기구 부착형 IMU 센서로 이 문제를 근본적으로 해결하며, 축적된 데이터를 AI가 분석하여 개인 맞춤 점진적 과부하를 자동으로 추천한다.

**[문제 2] 헬스장 기구 점유 문제 — "운동 아닌 시간이 기구를 잡고 있다"**

피크 시간대(오후 6시~9시) 인기 기구(벤치프레스, 스쿼트랙 등) 앞 평균 대기 인원 2.5명, 운동 시간의 30~40%가 대기에 소비된다. 그러나 기구 혼잡의 **진짜 원인**은 운동 자체가 아니라, 세트 사이에 스마트폰을 보며 시간을 보내는 **비운동 점유 시간**에 있다.

> **현실**: 세트 간 휴식 90초가 표준이지만, SNS 확인, 유튜브 시청, 웹툰 읽기 등으로 실제 휴식 시간이 5~10분으로 늘어나는 경우가 빈번하다. 기구 1대의 적정 사용시간(5세트 기준 약 10~13분)이 실제로는 20~30분으로 2~3배 초과되는 것이 헬스장 혼잡의 핵심 원인이다.

기존에는 이를 해결할 기술적 인프라가 전무했다. 헬스장 직원이 일일이 "기구 사용 시간을 지켜주세요"라고 안내하는 것은 비현실적이며, 회원 간 갈등의 원인이 되기도 한다.

국내 헬스장만 약 **11,100개**. 기구 점유 문제 해결은 **B2B 시장의 핵심 니즈**이다. 피카는 NFC 기반 기구 사용시간 컨트롤 시스템으로 이 문제를 기술적으로 해결한다.

**[문제 3] PT 비용 부담 — "개인 맞춤 피드백의 고비용 장벽"**

올바른 운동 프로그램과 개인 맞춤 가이드를 위한 퍼스널 트레이닝(PT)은 서울 기준 1회 5~7만 원, 월 12회 기준 **60~84만 원**으로 대다수에게 부담이 크다. PT 없이 독학으로 운동하는 이용자는 적절한 중량 설정, 부위별 균형, 과훈련 방지 등에서 체계적 피드백을 받을 수 없다.

피카는 AI 데이터 분석 기반 맞춤형 가이드를 제공하여, **월 9,900원**으로 PT의 핵심 가치인 "개인 맞춤 운동 추천"을 대체한다.

**[문제 4] 운동 지속성의 위기 — "3개월의 벽"**

헬스장 신규 회원의 약 50%가 6개월 이내 이탈, 연간 회원 유지율은 50~60%에 불과. 핵심 원인은 **눈에 보이는 성과의 부재**와 **개인 맞춤 피드백의 부족**이다.

피카는 데이터 기반 성취감(1RM 향상 추이, 볼륨 성장 그래프)과 **운동 마일리지 시스템**(보험료 인하 + 인앱 포인트)으로 "운동할수록 돌아오는 보상"을 설계하여, 3개월의 벽을 돌파하는 지속 동기를 제공한다.

### (2) 기존 솔루션의 구조적 한계

#### 센서 전용 방식의 한계

현재 시장에 존재하는 센서 기반 피트니스 트래커는 두 가지 형태이다.

**① 기구 교체형 (EGYM, Technogym)**

| 한계 | 상세 |
|------|------|
| **하드웨어 비용** | 헬스장 기구 전체 교체, 1대당 수백만 원 ~ 수억 원 규모 |
| **확장 속도** | 기구 설치 → 영업 → 운반 → 설치 → 교육 → 수개월 소요 |
| **헬스장 부담** | EGYM은 $7.5B(2026년 Playlist 합병) 규모이지만, 중소 헬스장에 부적합 |

**② 바벨/손목 착용형 (GymAware, RepOne, Enode, Fort)**

| 제품 | 방식 | 가격 | 한계 |
|------|------|------|------|
| GymAware | LPT 줄 방식 | $1,995 | 운동별 셋업 복잡, VBT 전문가용 |
| RepOne | LPT 줄 방식 | $299 | 바벨 전용, 머신 미지원 |
| Enode | 자석 부착 IMU | $329 | 바벨 전용, 머신 미지원 |
| Fort | 손목 밴드 | $289-349 | 2026 Q3 출시 예정, YC 투자 (⚠️ 주요 경쟁사) |
| Beast Sensor | 바벨 부착 IMU | $150 | **사실상 단종** (횟수 오인식 문제) |

> **Beast Sensor의 교훈**: "어떤 운동에서는 모든 횟수를 잡았고, 다른 운동에서는 하나도 못 잡았다. 고스트 랩(유령 횟수)이 계속 나왔다." 90% 정확도여도 나머지 10%의 오류 때문에 사용자는 결국 직접 세게 됨.

#### CV 전용 방식의 한계

| 한계 | 상세 |
|------|------|
| **카메라 촬영 부담** | 헬스장에서 스마트폰을 거치대에 고정해야 함 |
| **시야 막힘** | 주변 사람이 카메라 앞을 지나가면 인식률 급락 |
| **배터리 소모** | CV 실시간 처리는 대표적인 고전력 작업 |
| **심리적 불편** | "촬영 중"이라는 느낌 → 본인 + 주변인 모두 불편 |

**결론: 피카는 기구에 직접 부착하는 IMU 센서로 운동 횟수를 자동 카운팅하고, BLE 스마트 컨트롤러로 무게를 1초 만에 입력하는 하드웨어 솔루션이다.**

### (3) 기술 환경의 전환점: 저가 IMU 센서의 성숙

2025~2026년을 기점으로, 6축 IMU 센서(FSM 온칩, 저전력 550µA)가 소형화·저가화되면서 피트니스 기구에 직접 부착 가능한 수준에 도달했다.

**ST LSM6DSV IMU**: FSM(유한 상태 머신) 8개 내장 → 센서 자체가 "운동 중/대기 중" 판단 → MCU 불필요. 가격 **$1.48**(100개 기준) — 동급 경쟁 센서 대비 **3~4배 저렴**.

**Nordic nRF52840 BLE 5.0 SoC**와 결합하면 실시간 데이터 전송이 가능하며, 스마트 컨트롤러를 통해 운동 중 폰을 만질 필요 없는 Zero Touch UX를 구현할 수 있다.

> **이 기술 성숙은 "기구 부착 IMU로 정밀 횟수 카운팅 + BLE 컨트롤러로 1초 무게 입력"을 합리적 비용으로 제공하는 전환점이다.**

### (4) 페르소나 시나리오: "AIoT와 함께하는 운동"

> **이OO씨 (42세, IT기업 개발자, 운동 경력 2년)**
>
> **Before — 피카 없이:**
> 퇴근 후 헬스장, 오후 7시. 벤치프레스 앞에 두 명이 대기 중이다. 10분 기다려 자리를 잡았다. "지난주에 70kg 8회였나, 75kg 6회였나..." 기억이 안 난다. 기록 앱을 써봤지만 세트마다 폰 입력이 귀찮아 2주 만에 포기했다. 감으로 70kg을 잡는다. 세 번째 세트, 어깨에 통증. 폼이 잘못된 건지, 중량이 과한 건지 모른다. PT를 받고 싶지만 월 84만 원은 부담이다. 세트 사이에 핸드폰을 보다 보니 15분이 훌쩍 지나갔고, 뒤에 기다리는 사람과 눈치 싸움이 시작된다.
>
> **After — 피카와 함께:**
> 벤치프레스 옆에 붙어 있는 피카 센서를 NFC로 탭한다. 스마트 컨트롤러 화면에 "지난 세션: 벤치프레스 72.5kg × 8회 × 4세트, **추천: 75kg × 7회**"가 뜬다. 컨트롤러에서 [75kg] 버튼을 누른다(1초). 운동 시작. 기구에 붙은 LSM6DSV 센서가 자동으로 횟수를 세고, 컨트롤러 화면에 "1회... 2회... 3회..."가 표시된다. 세트 완료 → 휴식 타이머 자동 시작 → 폰은 건드릴 필요가 없다. 모든 데이터가 BLE로 앱에 자동 저장된다. **"4주간 1RM 82kg → 87kg 향상"** — PT 없이도, 데이터가 말해준다.
>
> NFC 탭 후 세트별 쉬는시간 카운터가 자동 작동하여, 정확한 90초 휴식 후 바로 다음 세트 진행. 불필요한 기구 점유 없이 효율적으로 운동을 마친다. 앱이 "이번 주 어깨 볼륨이 전체의 8%로 부족합니다. 사이드 레터럴 레이즈 2세트 추가를 추천합니다"라고 AI 피드백을 보낸다. 3개월 연속 주 3회 이상 운동 달성 → 보험사 마일리지 5,000포인트 적립 알림. 연간 보험료 2% 할인 혜택으로 연결된다.

| 대기 (FSM 경비 모드) | 12시간/일 | 0.030mA | 0.36mAh |
| 딥 슬립 | 6시간/일 | 0.005mA | 0.03mAh |
| MCU + BLE (FSM 덕분에 감소) | — | — | ~17mAh |
| **총 일일 소모** | | | **~66.4mAh** |
| **500mAh 기준 수명** | | | **약 7.5일** |

> ⚠️ OLED 디스플레이는 배터리 소모의 주요 요인이다 (일일 소모의 ~24%).
>
> **OLED 30초 자동 꺼짐 + FSM "운동 중에만 MCU 깨우기" 적용 시:**
> - OLED 일일 소모: ~16mAh → ~6mAh (30초 타임아웃)
> - 총 일일 소모: ~56.4mAh
> - **배터리 수명: 약 8~9일 (주 1회 충전 수준)**
>
> FSM 8개를 극한 최적화(운동 미감지 시 센서 자체 슬립)하면 **~10~12일**까지 연장 가능성이 있으나, 이는 실측 검증 후 확정할 수치이다.

### (3) 무게 입력 UX — 수동 입력 최적화 (Zero Touch 목표)

"운동 중 폰을 만질 필요 없음"이 핵심 UX 원칙이다.

**무게 입력 3가지 방법 (빠른 순)**

```
① 하드웨어 장치 직접 입력 (1초)
   → 기구 옆 스마트 컨트롤러에서 프리셋 버튼 탭
   → BLE로 앱에 즉시 전송

② 이전 기록 자동 팝업 (확인만, 0.5초)
   → NFC 탭 시 "지난번: 72.5kg × 8회 → 오늘 추천: 75kg × 7회" 자동 표시
   → 동일하면 확인 한 번

③ ±2.5kg 빠른 조정 (2초)
   → 하드웨어 장치 "+/-" 버튼으로 2.5kg 단위 조정
```

**UX 시나리오 (전체)**

```
1. 기구 도착 → NFC 탭 (1초)
   → 기구 자동 인식 + 이전 기록 팝업
   → "벤치프레스 | 지난번 72.5kg × 8회"

2. 무게 입력 (1초)
   → 컨트롤러에서 [75kg] 프리셋 버튼

3. 운동 시작
   → 기구 센서(LSM6DSV): 횟수 자동 카운팅 → 컨트롤러 화면 표시

4. 세트 완료
   → "10회 완료!" 자동 기록
   → 휴식 타이머 자동 시작

5. 전체 운동 종료
   → 운동 요약 자동 생성 (볼륨, 1RM, 부위별 분석)
```

### (3-1) NFC 기반 기구 사용시간 컨트롤 시스템

**목적**: 기구 점유 문제의 기술적 해결 — 불필요한 점유시간 방지 + B2B 회전율 극대화

**작동 흐름:**

```
[기구 사용시간 컨트롤 시스템]

1. 기구 도착 → NFC 탭 (1초)
   → 앱과 자동 연동 + 기구 사용 시작 기록
   → 해당 기구의 이전 사용 기록 팝업 (참고용)
   → 사용자별 운동 프로필에 따른 추천 세트/무게 표시

2. 운동 중 — 세트별 시간 관리
   → IMU 센서가 세트 완료 자동 감지
   → 매 세트 종료 시, 사용자 설정 쉬는시간 카운트다운 시작
   → 컨트롤러 화면: "휴식 01:30 → 01:29 → ..."
   → 휴식 종료 시 진동 알림: "다음 세트 준비!"

3. 총 기구 사용시간 자동 산출
   → 총 사용시간 = (세트 수 × 평균 운동시간) + (세트 수 × 쉬는시간) + 여유시간
   → 예시: 벤치프레스 5세트
     = 5세트 × 40초(운동) + 5세트 × 90초(휴식) + 2분(여유)
     = 약 13분

4. 시간 관리 알림
   → 80% 경과: "기구 사용시간 80% 경과" 안내
   → 100% 초과: "기구 사용시간 종료. 다음 회원에게 양보해주세요" 알림
   → B2B 대시보드: 기구별 실시간 사용 현황 + 대기열 표시
```

**핵심 효과:**

| 효과 | 상세 |
|------|------|
| **비운동 점유 방지** | 세트 간 핸드폰(유튜브, 웹툰, SNS) 사용으로 인한 불필요 점유 시간 감소 |
| **기구 회전율 증가** | 기구당 평균 사용시간 20~30분 → 10~15분으로 단축 (약 2배 회전율) |
| **회원 갈등 해소** | "사용시간이 끝났습니다" 시스템 알림 → 직원/회원 간 마찰 제거 |
| **B2B 핵심 가치** | 헬스장 운영자: 피크타임 대기시간 50% 감소 → 회원 만족도 증가 → 이탈률 감소 |
| **데이터 자산** | 기구별 사용 패턴 데이터 축적 → 헬스장 기구 배치/투자 최적화 |

> **B2B 킬러 기능**: 헬스장 운영자가 가장 원하는 것은 "트래픽 없는 쾌적한 운영"이다. NFC 기반 기구 사용시간 컨트롤은 피카가 단순 운동 기록 앱이 아닌, **헬스장 운영 솔루션**으로 포지셔닝되는 핵심 차별점이다.

### (3-2) AI 맞춤형 운동 가이드 시스템

**목적**: PT 비용의 1/6 가격(월 9,900원)으로 데이터 기반 개인 맞춤 운동 코칭 제공

**AI 분석 파이프라인:**

```
[운동 데이터 축적]
     │
     ▼
[AI 패턴 분석 엔진]
     ├── 점진적 과부하 추천: "지난 4주 벤치프레스 72.5kg→75kg→77.5kg 성장 → 이번 주 80kg × 6회 추천"
     ├── 부위별 균형 분석: "이번 달 가슴 볼륨 35%, 등 15% → 등 운동 비중 증가 필요"
     ├── 과훈련 경고: "이번 주 어깨 볼륨이 지난 주 대비 180% → 부상 위험, 휴식 권장"
     ├── 약점 부위 추천: "하체 운동 비중 12%로 전체 평균 25% 대비 부족 → 레그프레스 추가 추천"
     └── 1RM 예측: "현재 추정 1RM 92.5kg. 목표 100kg까지 약 6~8주 소요 예상"
```

**PT vs 피카 AI 코칭 비교:**

| 항목 | PT (퍼스널 트레이닝) | 피카 AI 코칭 |
|------|-------------------|-----------|
| 비용 | 월 60~84만원 | **월 9,900원** |
| 데이터 기반 | 트레이너 기억 의존 | **전 세션 데이터 자동 축적·분석** |
| 24시간 이용 | PT 시간만 | **언제든 앱에서 확인** |
| 점진적 과부하 | 트레이너 경험 기반 | **수치 기반 정밀 추천** |
| 부위 균형 | 주관적 판단 | **볼륨 데이터 기반 객관적 분석** |

> **핵심 가치**: "기록이 쌓일수록 AI가 더 정확해진다." 피카는 운동할수록 개인에게 최적화되는 AI 코칭 시스템이다.
