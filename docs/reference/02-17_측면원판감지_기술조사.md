# 측면 원판 감지 기술 조사 (2026-02-17)

> 정면 vs 측면 촬영 비교 + 업계 사례 + 권장 방향

## 결론 먼저

**바벨 원판 무게를 카메라로 자동 감지하는 상용 앱/서비스는 현재 존재하지 않음.**
Tempo조차 자체 색상 코딩 덤벨에서만 가능, 바벨 미지원.

---

## 현재 모델은 정면 전용

우리 YOLO26-N 모델: 바벨 끝(정면)에서 촬영한 537장으로 학습.
측면 데이터 없음 → 측면에서는 작동 안 함.

---

## 측면 촬영의 기술적 난이도

### 원판 측면 두께 (캐스트 아이언)

| 무게 | 두께 | 직경 |
|------|------|------|
| 20kg | 48~56mm | 450mm |
| 15kg | 41~48mm | 350~400mm |
| 10kg | 36~41mm | 270~310mm |
| 5kg | 23~30mm | 230~265mm |
| 2.5kg | 15~23mm | 210~215mm |

### 왜 어려운가

1. **극단적 종횡비**: 측면 원판 = 두께 50mm × 직경 450mm = 1:9 비율. YOLO bbox 학습 불안정.
2. **Small Object**: 640×640 입력에서 5kg 원판(25mm)은 5~10픽셀 너비.
3. **경계 구분 어려움**: 밀착된 원판 사이 간격이 거의 없음.
4. **라벨링 극도 어려움**: 얇은 원판 경계에 bbox 정확히 그리기 어려움.

### 범퍼 플레이트 문제

IWF 경기용 범퍼 플레이트는 10~25kg 전부 직경 450mm 동일.
측면에서 두께만으로 구별해야 함 → 8mm 차이로 20kg vs 15kg 구별.

## 정면 vs 측면 비교

| 항목 | 정면 (현재) | 측면 |
|------|-----------|------|
| 단일 원판 정확도 | 96.2% (실측) | 60~75% (추정) |
| 같은 무게 겹침 | **불가능** | 가능 (이론적) |
| 데이터 필요 | 537장 (완료) | 1,000~3,000장 (신규) |
| 라벨링 난이도 | 보통 | 매우 어려움 |
| 추가 개발 기간 | 0 | 4~8주 |
| UX 자연스러움 | 높음 | 낮음 (카메라 전환) |
| 사용 카메라 | 전면 | 후면 → 전면 전환 필요 |
| 같은 모델? | - | 별도 모델 필요 |
| 업계 사례 | 없음 | 없음 |

---

## 업계 사례

| 제품 | 방식 | 무게 감지 |
|------|------|----------|
| Tempo | 3D ToF 깊이 센서 | 자체 색상 코딩 덤벨만, 바벨 미지원 |
| BarbellCV | 고대비 마커 부착 | 속도만 추적, 무게 수동 입력 |
| SpaceO | YOLO 15,000장 | 바벨 위치만, 원판 무게 미지원 |
| Motra | 가속도 센서 | 모션 패턴, 무게 수동 |

---

## 하이브리드 가능성

정면(무게 종류) + 측면(개수) 조합:
- Stage 1A: 정면 → "20kg, 10kg 감지" (어떤 원판이 있는지)
- Stage 1B: 측면 → "원판 3장" (총 몇 장인지)
- 합산: 20+10+10 = 40kg per side

하지만 측면에서 "원판 개수 카운팅"만 해도 위 기술적 난이도는 동일.
