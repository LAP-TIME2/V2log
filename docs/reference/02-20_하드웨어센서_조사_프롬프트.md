# 하드웨어 센서 기반 횟수 카운팅 — 다른 AI 독립 조사용 프롬프트

> 이 문서는 ChatGPT, Gemini, Grok 등 다른 AI에게 동일한 주제의 독립 조사를 시키기 위한 배경설명 + 프롬프트입니다.
> Claude의 조사 결과와 비교·토론하여 더 나은 결론을 도출하는 것이 목적입니다.

---

## 배경설명 (이 부분을 먼저 AI에게 제공)

### 우리 회사 소개
- **회사명**: 피카(Fica) — 피트니스 스타트업 (예비창업 단계)
- **대표**: 이관주 (16년 피트니스 경력)
- **원래 방향**: IoT 기반 헬스장 자동 기록 시스템
- **피벗 배경**: 스마트폰 카메라 + AI(CV)로 무게 자동 감지를 시도했으나 실패 확인
  - YOLO (원판 인식): 단일 원판 OK, 겹친 원판 과소추정 (해결 불가)
  - MiDaS 깊이 추정: 단안 카메라 정확도 부족
  - Sobel 에지 감지: 노이즈에 취약
  - VLM (Gemini API): 최고 71%, 목표 80% 미달
  - **결론: 카메라로 무게 자동 감지는 포기**
- **CV 성과**: 횟수 카운팅은 MediaPipe BlazePose로 88~95% 달성 (이건 기술력 증명으로 활용)

### 새 방향: 하드웨어 센서
- 기구(머신, 바벨 등)에 **IMU 센서(가속도+자이로)**를 부착하여 횟수를 자동 카운팅
- BLE로 스마트폰 앱에 실시간 전송
- NFC 태그로 기구 식별 + 사용자 식별
- **무게 자동 감지는 하지 않음** — 무게는 앱에서 빠른 수동 입력

### 2025년 사업계획서의 원래 하드웨어 사양
- 센서: MPU6050 (6축 IMU)
- 장치 크기: 12cm × 12cm × 2cm
- 통신: BLE 5.0 + NFC
- 기구 인식: NFC 태그 (기구별 부착)
- 부착 위치: 바벨바, 케이블 손잡이, 머신 프레임

### 사업 모델
- **B2B**: 헬스장에 센서 장치 설치 → 회원들이 앱 사용
- **수익**: 헬스장 월 구독료 + 회원 프리미엄 구독
- **부가 기능**: 기구 트래픽 관리 (실시간 사용 현황), 회원 편의 (기구 예약, 대기열)
- **경쟁사**: EGYM (독일, 기구 제조사 직접 통합), Technogym (이탈리아, 고가), Strong/Hevy (앱만, 수동 입력)

## 프롬프트 (이 부분을 AI에게 질문으로 제공)

```
당신은 피트니스 하드웨어 + 소프트웨어 사업의 실현가능성을 조사하는 전문 컨설턴트입니다.

위 배경설명의 회사가 "헬스장 기구에 IMU 센서(가속도계+자이로스코프)를 부착하여 운동 횟수를 자동 카운팅하는 B2B 시스템"을 개발하려 합니다.

아래 6개 영역에 대해 각각 독립적이고 심층적으로 조사해주세요.
반드시 근거(논문, 제품, 데이터)를 포함하고, 추측이면 추측임을 명시해주세요.

---

### 조사 영역 1: 기존 제품 조사
- IMU 기반 운동 추적 상용 제품을 모두 찾아주세요 (PUSH Band, Atlas, Beast Sensor, GymAware, RepOne, Enode, Fort 등)
- 각 제품의: 센서 방식, 부착 위치, 정확도, 가격, 현재 상태(판매중/단종/인수)
- "기구 프레임에 직접 부착"하는 상용 제품이 있는지 특히 조사
- 실패한 제품들의 실패 원인 분석

### 조사 영역 2: 학술 연구
- IMU 기반 운동 횟수 카운팅 관련 논문을 찾아주세요
- 특히 "웨이트 스택/기구 프레임에 센서 부착" 관련 연구 (W8-Scope 등)
- 알고리즘별 정확도 비교: Peak Detection, Complementary Filter, CNN, LSTM
- 운동 종류별 정확도 차이

### 조사 영역 3: 센서 하드웨어
- MPU6050 vs BMI270 vs ICM-42688-P vs LSM6DSO32 비교
  - 노이즈 밀도, 드리프트, 온칩 ML 지원, 가격, 전력 소비
- 헬스장 환경(진동, 충격, 온도)에서의 내구성
- 센서 선정 추천과 근거

### 조사 영역 4: 운동 종류별 실현가능성
- 아래 운동 각각에 대해 IMU 센서로 횟수 카운팅이 가능한지 판정해주세요:
  - 바벨: 벤치프레스, 스쿼트, 데드리프트, OHP, 바벨로우
  - 머신: 레그프레스, 랫풀다운, 체스트프레스, 레그컬, 레그익스텐션, 케이블 로우
  - 스미스 머신
  - 풀업/딥스
  - 덤벨 운동 전체
- 각각 센서 최적 부착 위치와 예상 정확도 포함

### 조사 영역 5: 비즈니스 비판 (쇼스토퍼 찾기)
이 사업의 "죽일 수 있는 문제(쇼스토퍼)"를 찾아주세요:
- 다중 사용자 구분 문제 (같은 기구를 여러 명이 쓸 때)
- 덤벨/자유중량 미지원 문제 (헬스장 운동의 40~50%)
- 기구 100종 이상의 호환성 관리 문제
- 기구 제조사 보증 무효화 문제
- 배터리/충전 관리 문제 (50대 이상)
- 기존 앱(Strong, Hevy)의 수동 입력 대비 장점 부족 문제
- 스마트워치(Apple Watch, Galaxy Watch)의 운동 추적 기능과의 경쟁
- 헬스장 폐업률(5년 내 81.6%)과 B2B 고객 유지 문제
- 그 외 내가 놓친 쇼스토퍼가 있다면 추가

### 조사 영역 6: 대안 제안
위 조사를 종합하여:
- 이 사업이 성공하려면 어떤 조건이 필요한지
- 가장 현실적인 첫 진입 전략 (어디서, 어떤 기구부터, 어떤 고객에게)
- PT 스튜디오 vs 일반 헬스장 vs 홈짐 중 어디가 최적인지
- 센서 방식 외에 더 나은 대안이 있는지

### 출력 형식
- 각 영역별로 독립된 섹션으로 구분
- 표(table)를 적극 활용
- 근거가 있으면 출처(논문명, URL 등) 반드시 포함
- 추측인 경우 "⚠️ 추측" 표시
- 최종 종합 판정을 마지막에 포함: "GO / CONDITIONAL GO / NO-GO"와 조건
```

## 사용 방법

1. **배경설명** 부분을 먼저 붙여넣기
2. **프롬프트** 부분을 질문으로 입력
3. AI별로 독립 조사 결과를 받음
4. Claude 조사 결과(`02-20_하드웨어센서_횟수카운팅_실현가능성_조사보고서.md`)와 비교
5. 각 AI의 결론이 다른 부분을 중심으로 토론

### 비교 포인트 (토론 시 집중할 부분)
- [ ] MPU6050 → BMI270 교체 의견이 동일한가?
- [ ] 쇼스토퍼 목록이 일치하는가? 내가 놓친 게 있는가?
- [ ] 운동별 실현가능성 판정이 일치하는가?
- [ ] 첫 진입 전략 의견이 같은가? (PT 스튜디오 vs 일반 헬스장)
- [ ] "기구 프레임 부착 = 블루오션"이라는 판단에 동의하는가?
- [ ] Beast Sensor 실패 분석이 일치하는가?
- [ ] 최종 GO/NO-GO 판정이 같은가?
