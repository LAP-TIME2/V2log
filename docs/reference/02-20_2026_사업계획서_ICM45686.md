### 아이템 개요

피카(Fica)는 **기구 부착형 IMU 센서(ICM-45686) + BLE 스마트 컨트롤러**로 **운동 횟수를 자동 카운팅**하고, **무게를 1초 만에 입력**하는 AIoT 피트니스 자동 기록 플랫폼이다.

**핵심 기능:**
- **횟수 자동 카운팅**: ICM-45686 센서가 기구 진동 패턴을 직접 감지 → 운동 횟수·세트 자동 기록
- **무게 입력 1초 UX**: 기구 옆 전용 컨트롤러에서 프리셋 버튼 1회 → BLE 전송 → 폰을 꺼낼 필요 없음
- **스마트 기록 보조**: 이전 세션 기록 자동 팝업 + 점진적 과부하 추천

**기존 수동 기록 앱 vs 피카 IMU 센서:**

| 항목 | 기존 수동 기록 앱 | **피카 (IMU 센서)** |
|------|-----------------|-------------------|
| 횟수 기록 | 수동 입력 (매 세트) | **센서 자동 카운팅** |
| 무게 입력 | 폰 꺼내서 수동 입력 | **기구 옆 컨트롤러 1초 입력** |
| 배터리 | 앱만 → 낮음 | **센서 저전력 → ~10~12일** |
| 운동 중 조작 | 매번 폰 터치 | **Zero Touch (조작 0건)** |
| 확장성 | 앱 배포 즉시 | **앱 + H/W 패키지** |

> 피카는 단순 기록 앱이 아니라, **"월 9,900원으로 운동 기록을 완전 자동화하는 AIoT 플랫폼"**이다.

# 1. 문제인식 (Problem)

## 1-1. 창업아이템 배경 및 필요성

### (1) 급성장하는 피트니스 시장과 해결되지 않은 구조적 문제

글로벌 피트니스 시장은 2025년 약 **2,570억 달러(약 360조 원)** 규모이며, 헬스&피트니스 클럽 시장만 **1,256억 달러(CAGR 9.2%)** 로 성장 중이다 (Grand View Research, 2025). AI 피트니스 시장은 더욱 빠르게 성장하여 2024년 **98억 달러** → 2034년 **461억 달러** (CAGR 16.8%, InsightAce Analytic). 국내 피트니스 시장 역시 2025년 약 **47.5억 달러(약 6.3조 원)**, 스마트 피트니스 CAGR **19.6%**로 안정적 성장세를 보인다 (Mordor Intelligence).

그러나 이 거대한 시장의 이면에는 해결되지 않은 핵심 문제들이 존재한다.

**[문제 1] 운동 데이터의 수동 기록 — "기억에 의존하는 운동"**

헬스장에서 웨이트 트레이닝을 수행하는 대다수의 이용자는 운동 기록을 체계적으로 관리하지 못한다. 세트마다 폰을 꺼내 숫자를 입력하는 것은 운동 흐름을 깨뜨리며, 대부분 2주 만에 포기한다. "지난주에 벤치프레스를 몇 kg으로 했는지" 기억하지 못한 채 매번 감(感)에 의존하여 운동을 반복하는 것이 현실이다.

이는 **과학적 점진적 과부하(Progressive Overload)의 실패**로 직결된다. 기록 없이는 중량이나 횟수를 점진적으로 늘려나갈 근거가 없으며, 정체기 → 회의감 → 운동 중단의 악순환을 유발한다.

피카는 기구 부착형 ICM-45686 IMU 센서로 이 문제를 근본적으로 해결하며, 축적된 데이터를 AI가 분석하여 개인 맞춤 점진적 과부하를 자동으로 추천한다.

**[문제 2] PT 비용 부담 — "개인 맞춤 피드백의 고비용"**

올바른 자세와 개인 맞춤 운동 가이드를 위한 퍼스널 트레이닝(PT)은 서울 기준 **1회 5~7만 원, 월 12회 기준 60~84만 원**으로 대다수에게 부담이 크다. 피카는 IMU 센서 자동 기록 + **AI 맞춤형 코칭 시스템**으로 PT 의존도를 대폭 낮춘다. 운동 데이터가 축적될수록 AI 추천이 정교해지며, PT 1회(5~7만원) 비용의 1/6인 **월 9,900원**으로 점진적 과부하 추천, 부위별 균형 분석, 과훈련 경고 등 핵심 코칭 기능을 제공한다.

**[문제 3] 헬스장 기구 점유 문제 — "운동 아닌 시간이 기구를 잡고 있다"**

피크 시간대(오후 6시~9시) 인기 기구(벤치프레스, 스쿼트랙 등) 앞 평균 대기 인원 2.5명, 운동 시간의 30~40%가 대기에 소비된다. 그러나 기구 혼잡의 **진짜 원인**은 운동 자체가 아니라, 세트 사이에 스마트폰을 보며 시간을 보내는 **비운동 점유 시간**에 있다.

> **현실**: 세트 간 표준 휴식 시간은 90초이지만, SNS 확인, 유튜브 시청, 웹툰 읽기 등으로 실제 휴식이 5~10분으로 늘어나는 경우가 빈번하다. 기구 1대의 적정 사용시간(5세트 기준 약 10~13분)이 실제로는 20~30분으로 2~3배 초과되는 것이 혼잡의 핵심 원인이다.

기존에는 이를 해결할 기술적 인프라가 전무했다. 헬스장 직원이 "기구 사용 시간을 지켜달라"고 안내하는 것은 비현실적이며, 회원 간 갈등의 원인이 되기도 한다.

국내 헬스장만 약 **11,100개**. 기구 점유 문제 해결은 **B2B 시장의 핵심 니즈**이다. 피카는 NFC 기반 기구 사용시간 컨트롤 시스템으로 이 문제를 기술적으로 해결한다.

**[문제 4] 운동 지속성의 위기 — "3개월의 벽"**

헬스장 신규 회원의 약 50%가 6개월 이내 이탈, 연간 회원 유지율은 50~60%에 불과. 핵심 원인은 **눈에 보이는 성과의 부재**와 **개인 맞춤 피드백의 부족**이다.

피카는 데이터 기반 성취감(1RM 향상 추이, 볼륨 성장 그래프)과 **운동 마일리지 시스템**(보험료 인하 + 인앱 포인트)으로 "운동할수록 돌아오는 보상"을 설계하여, 3개월의 벽을 돌파하는 지속 동기를 제공한다.

### (2) 기존 솔루션의 구조적 한계

피카(Fica)는 창업 초기 IoT 센서 단독, CV 전용 등 다양한 접근법을 검토한 끝에, **기구 부착형 ICM-45686 IMU 센서 + BLE 스마트 컨트롤러** 구조를 채택하였다. CV 전용 방식은 거치대 부담, 배터리 소모, 시야 방해, 촬영 심리적 부담 등 헬스장 실환경 한계가 컸다.

피카는 기구에 직접 부착하는 ICM-45686 IMU 센서로 운동 횟수를 자동 카운팅하고, BLE 스마트 컨트롤러로 무게를 1초 만에 입력하는 하드웨어 솔루션이다.

### (3) 기술 환경의 전환점

2025~2026년을 기점으로, 6축 IMU 센서(ICM-45686, BalancedGyro™, 220µA 저전력)가 소형화되면서 피트니스 기구에 직접 부착 가능한 수준에 도달하였다. Nordic nRF52840 BLE 5.0 SoC와 결합하면 실시간 데이터 전송이 가능하다.

**IMU 센서 기술 혁신 — TDK ICM-45686:**
- **전력 220µA (6축 50Hz)**: 동급 센서 대비 2~3배 낮은 소비 전력 → 배터리 수명 극대화
- **충격 내성 20,000g**: 바벨 낙하 충격(1,000~5,000g) 대비 4배 여유 → 헬스장 내구성 최고
- **BalancedGyro™ 기술**: TDK 독자 특허 기술, 헬스장 진동 환경에서도 자이로 안정성 보장
- **FIFO 8KB**: MCU 슬립 280ms 유지 가능 → 배터리 추가 절약

> **이 기술 성숙이 "기구 부착형 센서로 운동을 자동 기록하는" 피카 비즈니스 모델을 실현 가능하게 만들었다.**

### (4) 페르소나 시나리오: "이OO씨의 변화"

> **이OO씨 (42세, IT기업 개발자, 운동 경력 2년)**
>
> **Before — 피카 없이:**
> 퇴근 후 헬스장, 오후 7시. 벤치프레스 앞에 두 명이 대기 중이다. 10분 기다려 자리를 잡았다. "지난주에 70kg 8회였나, 75kg 6회였나..." 기억이 안 난다. 세트마다 폰을 꺼내 기록 앱에 입력하려다 흐름이 깨져 포기했다. 감으로 70kg을 잡는다. 세트 사이에 핸드폰을 보다 보니 15분이 훌쩍 지나갔고, 뒤에 기다리는 사람과 눈치 싸움이 시작된다. 기록이 없으니 점진적 과부하도 안 되고, 3개월째 변화 없다.
>
> **After — 피카와 함께:**
> 벤치프레스에 도착. 기구 프레임에 부착된 피카 센서를 스마트폰으로 **NFC 탭 (1초)**. "이전 세션: 벤치프레스 72.5kg × 8회 × 4세트 / 추천: 75kg × 7회" 데이터 기반 가이드가 뜬다. 기구 옆 **피카 컨트롤러**에서 [75kg] 버튼 한 번 (1초). 스마트폰을 가방에 넣는다. 운동 시작. 기구 센서가 자동으로 횟수를 세고 컨트롤러 화면에 "1회... 2회..."가 표시된다. 세트 종료 — 휴식 타이머 자동 시작. NFC 탭 후 세트별 쉬는시간 카운터가 자동 작동하여, 정확한 90초 휴식 후 바로 다음 세트 진행. 불필요한 기구 점유 없이 효율적으로 운동을 마친다. 운동 종료 후 폰을 꺼내면 총 볼륨, 1RM 변화 추이가 자동 정리되어 있다. **"4주간 1RM 80kg → 87kg 향상"**. 폰을 한 번도 만지지 않고 운동을 완료했다.
>
> 앱이 "이번 주 어깨 볼륨이 전체의 8%로 부족합니다. 사이드 레터럴 레이즈 2세트 추가를 추천합니다"라고 AI 피드백을 보낸다. 3개월 연속 주 3회 이상 운동 달성 → 보험사 마일리지 5,000포인트 적립 알림. 연간 보험료 2% 할인 혜택으로 연결된다.

### (3) ICM-45686 센서 선정 근거

에이전트 3팀 독립 조사를 통해 4종 IMU 센서(BMI270, LSM6DSV, ICM-45686, ICM-42688-P)를 비교 분석한 결과, **TDK ICM-45686**을 선정하였다.

#### 4종 센서 비교표

| 항목 | Bosch BMI270 | ST LSM6DSV | **TDK ICM-45686** | TDK ICM-42688-P |
|------|-------------|------------|------------------|-----------------|
| **전력 (6축 50Hz)** | 685µA | ~550µA | **220µA ← 최저** | 880µA |
| **가격 (100개)** | $5.28 | **$1.48~2.49 ← 최저** | $5.86 | $4.67 |
| **충격 내성** | 3,000g | 10,000g | **20,000g ← 최강** | 5,000g |
| **FIFO** | 2KB | 4KB | **8KB ← 최대** | 2KB |
| **알려진 문제** | 자이로 드리프트 버그 | 온도 드리프트 | **없음** | 온도+스파이크 |
| **제품 상태** | 현역 | 현역 | **신제품 (주력)** | 퇴역 중 |
| **Zephyr 지원** | ◎ | △ (간접) | ○ | ◎ |
| **배터리 수명** (500mAh, OLED 포함) | ~6일 | ~8~9일 | **~10~12일** | ~5일 |
| **횟수 카운팅** | △ (드리프트) | ◎ | **◎** | △ (스파이크) |

#### ICM-45686 선정 이유 (3가지 핵심 강점)

**① BalancedGyro™ — 헬스장 환경 특화 (TDK 독자 기술)**
- 헬스장은 바벨 낙하, 기구 충격, 에어컨 진동 등 IMU에 최악의 환경
- BalancedGyro™는 진동이 많은 환경(드론, VR, 헬스장)에서도 자이로 안정성을 보장하는 TDK 독자 특허 기술
- 경쟁사 센서들이 모두 알려진 드리프트 버그를 가진 것과 달리, **ICM-45686은 알려진 버그 없음**

**② 전력 220µA × FIFO 8KB = 배터리 수명 극대화**
- 6축 50Hz 구동 시 220µA — 동급 센서 대비 2~3배 낮은 소비 전력
- FIFO 8KB: MCU를 280ms 슬립 유지 가능 (BMI270의 4배, ICM-42688-P의 4배)
- 센서 + MCU(nRF52840) + BLE 합산 실제 배터리 수명: **약 12일** (2주 목표 근접)
- OLED 30초 타임아웃 + FIFO 최적화 적용 시 **~10~14일** (실측 검증 필요)
- ⚠️ OLED(~20mA)가 배터리 소모의 주요 요인이므로, 실제 배터리 수명은 OLED 사용 패턴에 크게 의존

**③ 충격 내성 20,000g — 헬스장 내구성 최고**
- 바벨 낙하 시 발생 충격: 1,000~5,000g 수준
- ICM-45686의 20,000g은 최악의 헬스장 충격 대비 4배 여유
- 투자자 대상 "가장 오래가고 가장 튼튼한 피트니스 센서" 포지셔닝 가능

**리스크 및 대응:**
- 가격 프리미엄: LSM6DSV 대비 2~3배 비쌈 (50대 기준 ~26만원 추가) → 프리미엄 포지셔닝으로 판매가 15만원/대에 반영
- 커뮤니티 레퍼런스 제한: TDK 공식 Arduino 라이브러리 + Zephyr 바인딩으로 보완

### (3) 무게 입력 UX (수동 입력 최적화)

무게 자동 감지의 기술적 한계(단일 시점 RGB 카메라 기준 75% 상한)를 인식하고, **"운동 중 폰 불필요" 원칙** 하에 최적화된 수동 입력 UX를 설계하였다.

```
[무게 입력 UX 흐름]

방법 1: 피카 컨트롤러 (권장)
  기구 도착 → NFC 탭 (1초) → 컨트롤러에서 [프리셋 무게] 버튼 1회 → 완료

방법 2: 앱 빠른 입력
  이전 기록 자동 팝업 → ±2.5kg 버튼으로 조정 → 완료 (3초)

방법 3: 점진적 과부하 자동 제안
  "지난 세션 72.5kg × 8회 완료 → 오늘 추천: 75kg × 7회" 원탭 수락
```

**핵심 가치**: 이전 기록 자동 팝업 + ±2.5kg 버튼 + 프리셋 무게 버튼으로 **평균 3초 이내 입력** → "운동 중 폰을 만질 필요 없음" (폰은 가방 속에)

### (3-1) NFC 기반 기구 사용시간 컨트롤 시스템

**목적**: 기구 점유 문제의 기술적 해결 — 불필요한 점유시간 방지 + B2B 회전율 극대화

**작동 흐름:**

```
[기구 사용시간 컨트롤 시스템]

1. 기구 도착 → NFC 탭 (1초)
   → 앱과 자동 연동 + 기구 사용 시작 기록
   → 해당 기구의 이전 사용 기록 팝업 (참고용)
   → 사용자별 운동 프로필에 따른 추천 세트/무게 표시

2. 운동 중 — 세트별 시간 관리
   → ICM-45686 센서가 세트 완료 자동 감지
   → 매 세트 종료 시, 사용자 설정 쉬는시간 카운트다운 시작
   → 컨트롤러 화면: "휴식 01:30 → 01:29 → ..."
   → 휴식 종료 시 진동 알림: "다음 세트 준비!"

3. 총 기구 사용시간 자동 산출
   → 총 사용시간 = (세트 수 × 평균 운동시간) + (세트 수 × 쉬는시간) + 여유시간
   → 예시: 벤치프레스 5세트
     = 5세트 × 40초(운동) + 5세트 × 90초(휴식) + 2분(여유)
     = 약 13분

4. 시간 관리 알림
   → 80% 경과: "기구 사용시간 80% 경과" 안내
   → 100% 초과: "기구 사용시간 종료. 다음 회원에게 양보해주세요" 알림
   → B2B 대시보드: 기구별 실시간 사용 현황 + 대기열 표시
```

**핵심 효과:**

| 효과 | 상세 |
|------|------|
| **비운동 점유 방지** | 세트 간 핸드폰(유튜브, 웹툰, SNS) 사용으로 인한 불필요 점유 시간 감소 |
| **기구 회전율 증가** | 기구당 평균 사용시간 20~30분 → 10~15분으로 단축 (약 2배 회전율) |
| **회원 갈등 해소** | "사용시간이 끝났습니다" 시스템 알림 → 직원/회원 간 마찰 제거 |
| **B2B 핵심 가치** | 헬스장 운영자: 피크타임 대기시간 50% 감소 → 회원 만족도 증가 → 이탈률 감소 |
| **데이터 자산** | 기구별 사용 패턴 데이터 축적 → 헬스장 기구 배치/투자 최적화 |

> **B2B 킬러 기능**: 헬스장 운영자가 가장 원하는 것은 "트래픽 없는 쾌적한 운영"이다. NFC 기반 기구 사용시간 컨트롤은 피카가 단순 운동 기록 앱이 아닌, **헬스장 운영 솔루션**으로 포지셔닝되는 핵심 차별점이다.

### (5) AI 파이프라인 — ICM-45686 센서 기반

```
[ICM-45686 센서 데이터 (100Hz)]
     │
     ▼
┌──────────────────────────────┐
│ 1. Low-Pass Filter (5~10Hz)   │  노이즈 제거
│ 2. BalancedGyro™ 보정         │  진동 환경 안정화
│ 3. Peak Detection             │  반복 동작 패턴 감지
│ 4. Adaptive Threshold         │  운동 강도 자동 조절
│ 5. 횟수 판정 → BLE → 앱       │  ~280ms 지연 (FIFO 8KB)
└──────────────────────────────┘
             │
             ▼
    ┌────────────────┐
    │  V2log 앱 통합  │
    │  기록 저장      │
    │  통계 업데이트   │
    └────────────────┘
```

**FIFO 8KB의 실전 의미:**
- 센서가 100Hz로 데이터 생성 시 FIFO가 280ms치 데이터 저장
- MCU(nRF52840)는 280ms마다 한 번만 깨어나 데이터 수집 + BLE 전송
- BMI270(FIFO 2KB, 70ms) 대비 MCU 웨이크업 횟수 4분의 1로 감소 → 배터리 절약

> **B2B SaaS 핵심 가치**: 기구 사용시간 컨트롤 시스템이 제공하는 데이터는 헬스장 운영 최적화의 핵심이다. 기구별 평균 사용시간, 피크타임 대기 인원, 회원별 운동 패턴 등의 데이터를 웹 대시보드로 제공하여, 헬스장 운영자가 기구 배치, 운영 시간, 회원 관리를 데이터 기반으로 의사결정할 수 있게 한다.

> **ICM-45686 프리미엄 포지셔닝**: "가장 오래가고(~10~14일, 주 1회 미만 충전), 가장 튼튼한(20,000g) 피트니스 IoT 센서" → 15만원/대 프리미엄 가격 정당화. 경쟁사 대비 배터리 수명 + 내구성 우위 → TCO(총 소유 비용) 절감 소구점.
