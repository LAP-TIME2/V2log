# 겹친 웨이트 원판 장수/무게 자동 감지: 2026년 기술 현황 종합 조사 보고서

**날짜**: 2026-02-19
**목적**: 바벨에 겹쳐진 동일 웨이트 원판의 장수와 총 무게를 카메라로 자동 감지하는 기술의 현재 수준과 가능성을 철저히 조사
**핵심 질문**: "2D 카메라 1대로 겹친 원판 장수를 80% 이상 정확도로 알아내는 것이 현재 기술로 가능한가?"

---

## 핵심 결론 (Executive Summary)

> **2026년 현재, 단일 2D 카메라로 겹친 동일 웨이트 원판의 장수를 80% 이상 정확도로 자동 감지하는 것은 불가능하다.**

이것은 알고리즘의 한계가 아니라 **물리적/광학적 한계**이다. 원판 두께/지름 비율이 7-13%로 극히 작아서, 정면에서 1장과 2장의 시각적 차이가 bbox 지터, 렌즈 왜곡 등 잡음에 묻힌다. 전용 ToF 카메라 + 전용 컬러 원판을 사용하는 Tempo조차 동일 원판 겹침 감지에 실패했다.

**가장 유력한 대안 순위:**

| 순위 | 접근법 | 예상 정확도 | 비용 | 실현 시기 |
|------|--------|-----------|------|----------|
| 1 | **VLM (Gemini/GPT-4o) 측면 촬영** | 60-80% | 낮음 (API 비용) | 즉시 테스트 가능 |
| 2 | **Multi-Class YOLO (겹침 패턴 학습)** | 70-85% | 중간 (데이터 수집) | 2-4주 |
| 3 | **BLE 스마트 원판/바벨 센서** | 99%+ | 높음 (하드웨어) | 6개월+ |
| 4 | **현재 YOLO 클래스 식별 + 수동 장수** | ~100% | 없음 | 이미 구현됨 |

## 1. 상용 앱/제품 현황

### 1.1 Tempo (유일한 카메라 기반 무게 인식 제품)

**기술**: 3D Time-of-Flight (ToF) 깊이 카메라 + 전용 컬러코딩 원판
**가격**: $2,000+ (전용 스튜디오 장비)

**핵심 제한사항** (공식 FAQ 확인):
- "같은 색 원판을 2장 겹치지 마세요 -- 무게 인식이 감지할 수 없습니다"
- 바벨 지원 안 함 (덤벨만)
- 10lbs 이하만 인식
- Tempo 전용 원판만 인식 (일반 헬스장 원판 불가)

**시사점**: $2,000 전용 하드웨어 + 전용 ToF 카메라 + 전용 컬러 원판으로도 동일 원판 겹침 감지 불가. 이것은 단순한 소프트웨어 문제가 아님을 증명.

> 출처: [Tempo Weight Recognition FAQs](https://support.tempo.fit/support/solutions/articles/151000154718), [3D ToF Technology (Analog Devices)](https://www.analog.com/en/signals/articles/tempo.html)

### 1.2 Tonal

**기술**: 전자기 저항 방식 (물리적 원판 자체가 없음)
- 17개 센서로 움직임 추적
- 6축 자이로스코프 내장 스마트 핸들
- 무게 = 전자기장 제어 → 센서가 정확히 알고 있음 (카메라 감지 불필요)

**시사점**: 무게 감지 문제를 "원판을 없애는 것"으로 해결. 일반 헬스장에는 적용 불가.

> 출처: [Tonal 2](https://tonal.com/products/tonal-2), [Garage Gym Reviews](https://www.garagegymreviews.com/tonal-smart-home-gym-review)

### 1.3 EGYM Smart Strength

**기술**: 내장 로드셀/센서 + RFID 회원 인식
- 기계 자체가 현재 무게를 알고 있음 (pin-loaded 또는 전자 저항)
- 카메라 기반 무게 감지 아님
- 사용자별 자동 무게 조절

**시사점**: B2B 헬스장 장비 교체 방식. 기존 프리웨이트 바벨에는 적용 불가.

> 출처: [EGYM Smart Strength](https://us.egym.com/en-us/workouts/smartstrength)

### 1.4 VAY Sports (Nautilus 인수)

**기술**: RGB 카메라 + AI 동작 분석
- 운동 인식, 반복 카운팅, 폼 피드백에 특화
- **무게 감지 기능 없음** (자체적으로도 시도하지 않음)
- 2024년 Nautilus에 인수 → JRNY 플랫폼에 통합

**시사점**: 카메라 기반 피트니스 AI 전문 기업도 무게 감지는 시도조차 하지 않음.

> 출처: [VAY Sports](https://www.vay.ai/), [Nautilus acquires VAY](https://www.fittechglobal.com/fit-tech-news/Nautilus-acquires-motion-tech-specialist-VAY-as-it-looks-to-transform-its-digital-offer/348437)

### 1.5 Fitbod, RepCount, Strong, Hevy 등 피트니스 앱

- 모두 **수동 무게 입력** 방식
- AI는 루틴 추천/무게 증량 제안에만 사용
- 카메라 기반 무게 감지 기능 제공 앱: **0개**

- 무게는 여전히 수동 입력

| 제품 | 무게 감지 방식 | 겹친 원판 | 일반 헬스장 |

**결론**: 2026년 현재, 일반 헬스장에서 카메라로 겹친 원판 무게를 자동 감지하는 상용 제품은 **전 세계에 0개**이다.

## 2. 학술 논문 / 연구 현황

### 2.1 "Counting Stacked Objects" (Dumery et al., ICCV 2025 Oral)

**가장 직접적으로 관련된 논문.**

**방법론**:
- 3D Gaussian Splatting으로 기하 복원
- SAM2로 대상 영역 세그멘테이션
- DinoV2 인코더 + CNN 디코더로 점유 비율 예측
- 공식: `Count = (점유율 x 총 부피) / 단위 물체 부피`

**필요 조건**:
- **30-60장의 다중 뷰 이미지** (단일 뷰 아님!)
- 사용자가 첫 이미지에서 대상 영역 선택 필요
- 물체가 "대략 동일"하고 "균일하게 적재"되어야 함

**정확도**:
- 합성 데이터: sMAPE 27.65%
- 실제 데이터: sMAPE 34.20% (약 66% 정확도)

**한계**:
- 복잡한 형상에서 실패 (파스타, 핀 등)
- 단일 뷰로는 부피 추정 불가능
- 실시간 처리 불가 (30-60장 촬영 + 3D 복원 필요)
- **헬스장 원판에 적용 시**: 30-60장 촬영 = 비현실적

**시사점**: 최첨단 연구조차 단일 뷰가 아닌 다중 뷰를 사용하며, 그래도 66% 수준. ICCV Oral 논문이므로 이 분야 최고 수준의 결과임.

> 출처: [arxiv.org/abs/2411.19149](https://arxiv.org/abs/2411.19149), [GitHub: CorentinDumery/3d-counting](https://github.com/CorentinDumery/3d-counting)

### 2.2 산업용 적층 기판 계수 (Zhao & Dai, IEEE 2017; Optica 2020)

**방법론**: 통제된 측면 촬영 + 균일 조명 + 엣지 검출

**필요 조건**:
- 통제된 측면 촬영 (고정 카메라, 고정 거리)
- 통제된 조명 (균일 백라이트)
- 균일한 두께 (마이크로미터 단위 정밀도)
- 깔끔한 엣지 (산업용 절단면)

**정확도**: 99.99% (산업 환경)

**헬스장 적용 불가 이유**:
- 고정 카메라/거리/조명 불가
- 원판 엣지 불규칙 (손잡이 구멍, 각인, 고무 코팅)
- 배경 복잡 (다른 사람, 기구)

> 출처: [IEEE 2017](https://ieeexplore.ieee.org/document/8115255/), [Optica 2020](https://opg.optica.org/josaa/abstract.cfm?URI=josaa-37-7-1206)

### 2.3 barbellcv (GitHub 오픈소스)

**기능**: 바벨 궤적 추적 + 속도 기반 훈련 (VBT)
- YOLOv5로 바벨 감지 → 궤적 추적
- **무게는 사용자 수동 입력** (카메라 감지 아님)
- 원판 종류/장수 식별 기능 **없음**

**시사점**: CV 기반 바벨 프로젝트도 무게 감지는 수동에 의존.

> 출처: [GitHub: tlancon/barbellcv](https://github.com/tlancon/barbellcv)

### 2.4 Olympic Weightlifters Performance Assessment (IEEE 2023)

- 관절 각도, 바벨 궤적, 리프팅 속도 분석
- **무게 감지**: 해당 없음 (경기에서 무게는 이미 알고 있음)

> 출처: [IEEE Xplore](https://ieeexplore.ieee.org/document/10302649/)

## 3. VLM (Vision-Language Model) 활용 가능성

### 3.1 VLM 카운팅 능력의 근본적 한계

**"Your VLM Can't Even Count to 20" (ICLR 2026 제출)**

핵심 발견:
- VLM은 **기억에 의존**하여 카운팅 (실제로 "세는" 게 아님)
- 4개 다리인 개 사진: 100% 정확 → 5개 다리로 수정된 개 사진: 여전히 "4개"라고 답변
- 수정된 이미지에서 평균 정확도 **17.05%**
- "무시하고 직접 세라"는 프롬프트로 개선: **+1.87%p** (거의 효과 없음)

> 출처: [OpenReview: VLMCountBench](https://openreview.net/forum?id=5JN68XdDli), [arxiv.org/abs/2510.04401](https://arxiv.org/abs/2510.04401)

**"Exposing the Failures of VLMs in Compositional Counting" (2025)**

- 단일 유형 물체 카운팅: 비교적 양호
- **복합 유형 카운팅** (여러 종류 섞임): 심각한 실패
- 기본 기하학 도형으로 테스트해도 실패 → 원판처럼 복잡한 물체는 더 어려움

**LVLM-COUNT (2025)**

- GPT-4o, Qwen2, Gemma 3 평가
- 최고 성능 모델(Qwen2.5 72B): Level 1에서 60%, Level 3에서 45%
- "분할 정복" 방식으로 개선 시도 → 일부 향상되지만 근본적 한계 존재

### 3.2 VLM으로 겹친 원판 세기: 현실적 시나리오 분석

**가능한 접근법**: 바벨 측면 사진을 VLM에게 보여주고 "몇 장인가?" 질문

**예상 시나리오별 정확도 추정**:

| 시나리오 | 예상 정확도 | 근거 |
|----------|-----------|------|
| 서로 다른 색 원판 (IWF 컬러) | 70-85% | 색 차이가 시각적 단서 |
| 동일 색 원판 1-2장 | 50-70% | 두께 차이 미미, 추측 수준 |
| 동일 색 원판 3장+ | 30-50% | 틈새 거의 안 보임 |
| 검정 원판 겹침 (일반 헬스장) | 30-60% | 가장 어려운 케이스 |

**VLM의 잠재적 장점**:
1. **세계 지식 활용**: "20kg 원판은 약 5cm 두께"를 이미 알고 있음
2. **컨텍스트 추론**: 바벨 슬리브 길이 대비 원판 스택 길이로 장수 추론 가능
3. **OCR 겸용**: 보이는 숫자/텍스트도 함께 읽기 가능
4. **Few-shot 학습**: "이 헬스장 원판은 이렇게 생겼다" 참조 이미지 제공 가능

**VLM의 한계**:
1. **Hallucination**: 실제로 세지 않고 "그럴듯한 답" 생성
2. **일관성 부족**: 같은 사진에 같은 질문해도 다른 답
3. **API 지연**: 1-3초 (실시간에는 부적합, 세트 간 휴식에는 가능)
4. **비용**: 이미지 1장당 약 258 토큰 (Gemini) / 가변 (GPT-4o)

### 3.3 VLM 접근법 평가

**현실적 기대치**:
- 측면 촬영 + 적절한 프롬프트로 **60-80%** 정확도 기대
- "확실한 경우만 자동 입력, 불확실하면 수동" 전략이면 **사용 가치 있음**
- 실시간은 불가, 세트 간 휴식 시 1장 촬영 → 1-3초 후 결과

**테스트 필요 사항**:
- Gemini 2.5 Flash / GPT-4o / Claude Vision 비교
- 측면 vs 정면 vs 대각선 촬영 각도별 정확도
- 프롬프트 엔지니어링 (바벨 슬리브 길이 힌트 제공 등)
- 동일 사진 10회 반복 시 일관성

> 출처: [DataCamp VLM 가이드](https://www.datacamp.com/blog/top-vision-language-models), [OpenAI 카운팅 문제](https://community.openai.com/t/gpt4-vision-model-counting-problem/691309), [Roboflow GPT-4V 실험](https://blog.roboflow.com/gpt-4v-object-detection/)

## 4. 측면 촬영 접근법

### 4.1 이론적 근거

바벨 측면에서 보면 원판들이 "나란히" 보이므로, 원판 사이 틈새나 두께 차이로 장수를 셀 수 있다는 가설.

**원판 물리 치수 (IWF 표준)**:

| 원판 | 지름 (mm) | 두께 (mm) | 1280px 촬영 시 두께 픽셀 |
|------|----------|----------|----------------------|
| 20kg | 450 | 33-52 | 약 47-75px (1m 거리) |
| 15kg | 375 | 34-45 | 약 49-65px |
| 10kg | 290 | 34-39 | 약 49-56px |
| 5kg | 248 | 22-27 | 약 32-39px |
| 2.5kg | 213 | 16-19 | 약 23-27px |

### 4.2 측면 촬영의 장점

1. **원판 두께가 직접 보임**: 정면 촬영 대비 정보량 10-50배 증가
2. **원판 간 틈새 감지 가능**: 완전 밀착이 아닌 경우 엣지 구분 가능
3. **바벨 슬리브 길이 = 기준자**: 41cm 슬리브 대비 스택 높이로 장수 추론

### 4.3 측면 촬영의 현실적 문제

1. **사용자에게 측면 촬영 요구** = 추가 조작 (무조작 원칙 위반)
2. **각도 일관성 없음**: 정확한 90도 측면이 아니면 투영 왜곡
3. **부분 가림**: 손, 랙, 벤치에 가려짐
4. **바벨 기울기**: 5도만 틀어져도 원판 겹쳐 보임
5. **원판 밀착 시**: 검정 원판끼리 완전 밀착하면 틈새 0 → 구분 불가

### 4.4 V2log 이전 테스트 결과 (Mode C: Sobel Edge Projection)

- Sobel 엣지 검출 → 히스토그램 피크 카운팅
- **결과: 실전 불가** (헬스장 테스트에서 실패)
- 원인: 배경 엣지, 손잡이 구멍, 고무 코팅 반사로 피크 오검출

### 4.5 측면 촬영 결론

이론적으로는 가장 정보량이 많지만, 실용성(사용자 조작 필요, 환경 통제 불가)과 정확도(검정 원판 밀착 시 구분 불가) 모두 문제. **VLM + 측면 촬영** 조합이 가장 유력한 절충안.

## 5. 멀티뷰 접근법

### 5.1 학술 연구 (Dumery et al., ICCV 2025)

위 §2.1에서 상세 기술. 30-60장 다중 뷰 필요 → 헬스장에서 비현실적.

### 5.2 2장 촬영 (정면 + 측면)

**시나리오**: Stage 1에서 정면 촬영 → 클래스 식별, 이후 측면 촬영 요청

- 정면: YOLO로 원판 종류 식별 (96% mAP)
- 측면: 두께 기반 장수 추정

**문제**: 사용자에게 2번 촬영 요구 = 무조작 원칙 위반
**가능한 완화**: 운동 시작 전 바벨 셋업 시 자연스럽게 2각도 촬영 유도

### 5.3 비디오에서 자동 다중 뷰

**시나리오**: 사용자가 바벨 근처를 이동하는 동안 비디오에서 다양한 각도 자동 캡처

- Structure from Motion (SfM)으로 3D 복원
- 실시간 처리 불가 (수십 초~수 분)
- 사용자가 의도적으로 "바벨 주위를 돌아야" 함

**결론**: 비현실적.

## 6. 세그멘테이션 기반 접근법

### 6.1 SAM2 (Segment Anything Model 2)

**능력**:
- 이미지/비디오에서 임의 객체 정밀 세그멘테이션
- 프롬프트 기반 (포인트, 박스, 텍스트)
- 비디오에서 객체 추적 (메모리 뱅크 기반)

**겹친 원판에 대한 한계**:
- SAM2는 **가시적 경계**를 기준으로 세그멘테이션
- 동일 색 원판이 밀착되면 경계가 없음 → 하나의 덩어리로 인식
- Auto 모드에서 작은/가려진 구조물 recall 낮음
- "원판 3장"이라는 사전 지식 없이는 분리 불가

### 6.2 SAM3 (2025.11)

- SAM2 대비 텍스트 프롬프트 지원 추가
- 오픈 보캐뷸러리 세그멘테이션
- 그러나 **물리적으로 겹친 동일 객체 분리** 문제는 여전히 미해결

### 6.3 Instance Segmentation의 근본적 한계

Instance Segmentation은 "시각적으로 구분 가능한 개별 객체"를 분리하는 기술. 동일 색 원판이 밀착되어 **시각적으로 하나의 객체**로 보이면, 어떤 세그멘테이션 모델도 분리할 수 없다. 이것은 알고리즘이 아니라 입력 데이터(이미지)의 한계.

> 출처: [Meta SAM2](https://ai.meta.com/sam2/), [SAM2 GitHub](https://github.com/facebookresearch/sam2)

## 7. 물리적 단서 활용

### 7.1 원판 두께 + 바벨 슬리브 길이 기반 추론

**알려진 상수**:
- 올림픽 바벨 슬리브 길이: 남자용 415mm, 여자용 320mm
- 20kg 원판 두께: 33-52mm (제조사마다 다름)
- 15kg: 34-45mm, 10kg: 34-39mm, 5kg: 22-27mm, 2.5kg: 16-19mm

**이론적 추론**:
```
바벨 슬리브 길이 (415mm) 중 점유 길이를 측정
→ 점유 길이 / 원판 두께 = 장수
예: 점유 길이 150mm ÷ 45mm/장 = 3.3장 → 3장
```

**현실적 문제**:

1. **두께 편차가 크다**: 같은 20kg도 33-52mm (57% 편차). 헬스장마다 다른 제조사 원판 사용.
2. **슬리브 길이 측정이 어렵다**: 2D 이미지에서 원근 왜곡 + 부분 가림
3. **밀착 vs 느슨**: 원판 사이에 갭이 있으면 점유 길이 증가
4. **바벨 칼라**: 양쪽 끝 고정 칼라가 슬리브 일부 점유
5. **촬영 각도 의존**: 정면에서는 두께가 거의 안 보임, 측면에서만 유효

### 7.2 Depth Estimation 기반 두께 측정

**Apple Depth Pro (2024-2025)**:
- 단안 RGB에서 메트릭 깊이 추정
- 0.3초 내 2.25메가픽셀 깊이 맵 생성
- 경계 정확도에서 기존 모델 대비 "배수 차이" 개선

**원판 두께 측정에 대한 한계**:
- 메트릭 깊이 정확도: 수 cm 수준 (mm 단위 불가)
- 원판 두께 16-52mm → cm 단위 정밀도로는 1장 vs 2장 구분 불가
- 반사 표면(금속, 고무) → 깊이 추정 불안정
- V2log MiDaS v2.1 테스트 (Mode B): **실전 불가** 판정

### 7.3 V2log 이전 테스트 결과

| 접근법 | 물리적 단서 | 결과 |
|--------|-----------|------|
| Mode A (Aspect Ratio) | bbox 가로/세로 비율로 두께 추정 | 20kg 3/4 성공, 소형 원판 실패 |
| Mode B (MiDaS Depth) | 단안 깊이로 스택 두께 측정 | 실전 불가 |
| Mode C (Sobel Edge) | 엣지 피크로 원판 경계 카운팅 | 실전 불가 |

## 8. 대안 기술 비교 분석

### 8.1 VLM 측면 촬영 (가장 유력한 단기 대안)

**개념**: 세트 시작 전 바벨 측면 사진 1장 → VLM API로 분석

**장점**:
- 개발 비용 매우 낮음 (API 호출만)
- VLM의 세계 지식(원판 두께, 바벨 규격) 자동 활용
- OCR + 카운팅 + 클래스 식별 동시 수행 가능
- 모델 업그레이드 시 정확도 자동 개선

**단점**:
- 네트워크 필요 (오프라인 불가)
- 1-3초 지연
- 응답 일관성 부족 (같은 사진에 다른 답)
- Hallucination 위험
- API 비용 (이미지당 ~$0.003-0.01)

**예상 정확도**: 60-80% (측면 촬영 + 최적화된 프롬프트)

**구현 난이도**: 낮음 (1-2일)

### 8.2 Multi-Class YOLO (겹침 패턴 학습)

**개념**: "20kg_x1", "20kg_x2", "20kg_x3" 등 겹침 패턴을 별도 클래스로 학습

**장점**:
- 오프라인 동작
- 실시간 처리 (30fps)
- 결정론적 (같은 입력 → 같은 출력)
- 기존 YOLO 파이프라인에 통합 용이

**단점**:
- 대량 데이터 필요 (클래스 수 × 각도 × 조명 = 수천 장)
- 클래스 폭발: 5무게 × 8장수 = 40 클래스
- 새 헬스장 원판에 일반화 어려움
- 학습/업데이트 비용

**예상 정확도**: 70-85% (충분한 데이터 확보 시)

**구현 난이도**: 중간 (2-4주, 데이터 수집 포함)

### 8.3 BLE 스마트 센서

**개념**: 원판이나 바벨에 무게 센서/RFID 부착

**종류**:
1. **스마트 바벨 칼라**: 바벨 끝에 끼우는 장치, 무게 자동 측정
2. **RFID 태그 원판**: 각 원판에 태그, 리더기로 스캔
3. **Smart Weight Pin**: 핀 로디드 기구용 자동 무게 추적

**장점**: 99%+ 정확도, 원판 장수/종류 완벽 식별
**단점**: 하드웨어 비용, 헬스장 설치 필요, B2B만 가능
**시사점**: V2log B2B 전략에는 유효, B2C 앱에는 부적합

> 출처: [GAO RFID 피트니스](https://gaorfid.com/sports-and-fitness-industry-using-ble/), [Stormotion BLE 가이드](https://stormotion.io/blog/how-to-integrate-ble-fitness-devices-into-app/)

### 8.4 하이브리드: YOLO 클래스 + VLM 장수

**개념**: YOLO로 원판 종류 빠르게 식별 → VLM으로 장수 정밀 확인

**장점**:
- YOLO 강점(실시간 클래스 식별 96%)과 VLM 강점(추론 능력) 결합
- YOLO가 실패하는 겹침 케이스에서만 VLM 호출 → API 비용 절감

**단점**:
- 2단계 처리로 복잡도 증가
- VLM 지연(1-3초)은 여전히 존재
- 두 모델의 실패 모드가 다를 수 있음

## 10. 핵심 질문에 대한 최종 답변

### "2D 카메라 1대로 겹친 원판 장수를 80% 이상 정확도로 알아내는 것이 현재 기술로 가능한가?"

**단답: 불가능 (정면 촬영 기준) / 조건부 가능 (측면 촬영 + VLM)**

**상세 답변:**

1. **정면 촬영**: 불가능. 원판 두께/지름 비율 7-13%로 시각적 차이가 잡음에 묻힘. Tempo도 ToF 카메라로 실패. 이것은 SW 문제가 아닌 물리적 한계.

2. **측면 촬영 + VLM**: **60-80%** 도달 가능성 있음. VLM의 세계 지식(원판 두께, 바벨 규격) + 시각 정보를 결합하면, 특히 2-3장 수준에서는 합리적 정확도 기대. 단, 80% "이상"을 안정적으로 달성하려면 상당한 프롬프트 최적화와 실증 테스트 필요.

3. **Multi-Class YOLO**: 충분한 데이터(겹침 패턴별 수백 장)를 확보하면 **70-85%** 가능. 그러나 데이터 수집 비용과 새 헬스장 일반화 문제.

4. **하이브리드 (YOLO 클래스 + VLM 장수)**: 가장 현실적인 **80% 도달 경로**. YOLO가 원판 종류를 정확히 식별(96%)하고, 겹침이 의심되면 VLM에 장수 확인을 위임.

### 권장 로드맵

```
[즉시] VLM 테스트 (V2log-VLM 프로젝트)
  └→ Gemini 2.5 Flash / GPT-4o로 측면 촬영 사진 정확도 측정
  └→ 목표: 60% 이상이면 앱 통합 진행

[2주 내] YOLO + VLM 하이브리드 프로토타입
  └→ YOLO: 원판 종류 + 단일 원판 무게 (기존 96%)
  └→ VLM: 겹침 감지 시 장수 추론 (1-3초 API)
  └→ Fallback: VLM 확신 없으면 YOLO 단일 결과만 사용

[1개월+] Multi-Class YOLO (옵션)
  └→ VLM 테스트 결과 60% 미만이면 대안으로 전환
  └→ 겹침 패턴 데이터 대량 수집 필요

[6개월+] B2B BLE 센서 (별도 라인)
  └→ 하드웨어 파트너십 필요
  └→ 99%+ 정확도, 프리미엄 서비스
```
