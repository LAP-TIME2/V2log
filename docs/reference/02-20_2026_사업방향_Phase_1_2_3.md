## 배경: 왜 방향을 재설정하는가

### CV 무게 감지의 현실

**4가지 접근법을 전수 테스트한 결과:**

| 접근법 | 원리 | 최고 정확도 | 판정 |
|--------|------|-----------|------|
| Aspect Ratio | bbox 종횡비로 두께 역산 | **75%** | 현재 최고, 업계 최초 |
| MiDaS Depth | 단안 깊이 추정 | 크래시 | 폐기 |
| Sobel Edge | 에지 히스토그램 피크 | 1→3-4장 오판 | 폐기 |
| VLM (Gemini) | LLM 직접 판단 | 71% | 폐기 |

**3대 AI (Gemini/ChatGPT/Opus) 독립 딥리서치 결론:**
- 일반 스마트폰 RGB 카메라, 단일 시점, 비통제 환경에서 겹친 원판 장수 80%+ 자동 감지는 **물리적으로 극히 어렵다**
- 사진 1장에 담기는 정보량 = **2 bits 이하** (겨우 2개 상태 구분 가능)

### Tempo 사례 — 업계가 증명한 한계

**Tempo** (미국 홈짐 스타트업, $17.5M Series A, Microsoft 파트너십):
- **전용 ToF(깊이) 카메라** + **전용 컬러코딩 원판** 사용
- 공식 FAQ: *"Try to avoid stacking duplicate plates... as our weight recognition cannot detect that."*
- **결론: $17.5M 투자 + 전용 하드웨어로도 겹친 원판 감지를 포기**

### 피카의 성과 — 업계 최초

> Tempo가 전용 ToF + 전용 컬러 원판으로도 포기한 겹친 원판 감지를,
> 피카는 **일반 스마트폰 전면 카메라만으로 Aspect Ratio 방식을 통해 20kg 원판 2장까지 감지에 성공**.
> 이는 공개적으로 확인된 업계 최초의 성과.

## Phase 1: 스마트 기록 플랫폼 (출시 단계)

**핵심 컨셉**: CV 포즈 분석 + 하드웨어 입력 장치

### 1-1. 횟수 자동 카운팅 (CV, 이미 개발 완료)

| 항목 | 현황 |
|------|------|
| 기술 | MediaPipe BlazePose 33개 관절 + One Euro Filter + Velocity Gate |
| 정확도 | **88~95%** |
| 지원 운동 | 90개 (8개 카테고리) |
| 실행 환경 | 스마트폰 온디바이스 (서버 전송 없음, 개인정보 보호) |
| 상태 | **실기기 테스트 완료** |

### 1-2. 무게 입력 하드웨어 장치 (신규 개발)

**왜 하드웨어인가?**
- CV 무게 자동 감지가 물리적 한계에 도달 (75% 천장)
- 스마트폰 수동 입력은 운동 흐름을 깨뜨림 (폰을 꺼내야 함)
- **운동 기구 바로 옆에서 1초 만에 입력할 수 있는 전용 장치** 필요

**하드웨어 장치 개요:**

```
┌──────────────────────────────────────────┐
│          피카 스마트 컨트롤러              │
│                                          │
│  ┌────────────────────────────────────┐  │
│  │  [무게 입력]  20kg × 2 = 40kg     │  │  ← 무게 빠른 입력 (프리셋 버튼)
│  │                                    │  │
│  │  [세트]  3/5 세트                  │  │  ← 현재 세트 / 목표 세트
│  │  [횟수]  8/10 회                   │  │  ← CV 자동 카운팅 결과 표시
│  │                                    │  │
│  │  [휴식]  01:23 / 01:30             │  │  ← 휴식 타이머 + 남은 시간
│  │                                    │  │
│  │  ████████████░░░  진행률 60%       │  │  ← 오늘 운동 진행 상황
│  └────────────────────────────────────┘  │
│                                          │
│  통신: BLE (Bluetooth Low Energy)        │
│  전원: 충전식 배터리 (USB-C)              │
│  부착: 자석/클립 (기구 옆에 부착)         │
│  크기: 약 80×50mm (카드 절반 크기)        │
└──────────────────────────────────────────┘
```

**입출력 기능:**

| 기능 | 방향 | 설명 |
|------|------|------|
| 무게 입력 | **입력** | 프리셋 버튼 또는 +/- 버튼으로 무게 빠르게 설정 |
| 세트 진행 | **출력** | 현재 세트 번호 / 목표 세트 표시 |
| 횟수 표시 | **출력** | 스마트폰 CV가 카운팅한 횟수 실시간 표시 |
| 휴식 타이머 | **출력** | 세트 종료 후 자동 시작, 남은 시간 표시 |
| 알림 | **출력** | 휴식 끝 진동/소리 알림, 세트 시작 안내 |
| 운동 진행률 | **출력** | 오늘 루틴 전체 진행 상황 |

**UX 시나리오:**

```
사용자 행동                          시스템 반응
────────────────────────────────────────────────────
1. 바벨에 20kg 원판 2장 끼움
2. 하드웨어 장치에서 [20kg] [×2] 탭    → 핸드폰에 40kg 자동 전달
   (1초, 손가락 2번)                      (BLE 통신)
3. 벤치프레스 시작                      → 폰 카메라가 자동 횟수 카운팅
                                        → 장치에 "1회... 2회... 3회..." 표시
4. 10회 완료, 바벨 내려놓음             → "10회 완료!" 장치에 표시
                                        → 휴식 타이머 자동 시작 "01:30"
5. 휴식 중 장치만 봄                    → 남은 시간 "00:45... 00:44..."
   (폰은 계속 카메라 촬영 중)
6. 진동 알림 "세트 시작!"               → 다음 세트 시작
```

**핵심 가치:**
- 운동 중 **폰을 만질 필요 없음** (폰은 카메라 역할만)
- 무게 입력이 **1초** (프리셋 버튼)
- 모든 정보가 **눈앞에** (폰 화면 안 봐도 됨)

### Phase 1 산출물

| 산출물 | 상세 |
|--------|------|
| 앱 (iOS/Android) | CV 횟수 카운팅(90개 운동) + 운동 기록 + 통계 |
| 하드웨어 장치 | BLE 스마트 컨트롤러 (무게 입력 + 정보 표시) |

## Phase 2: 하드웨어 기반 횟수 측정 (CV 보완)

**핵심 컨셉**: 스마트폰 CV → 기구 직접 센서로 전환

### 2-1. 왜 하드웨어 횟수 측정인가

| 문제 | 설명 |
|------|------|
| **공간 점유** | 스마트폰을 거치대에 고정해야 함 → 거치대 설치 번거로움 |
| **카메라 시야** | 주변 사람이 가리거나, 각도가 안 맞으면 인식률 하락 |
| **배터리** | CV 실시간 처리 = 배터리 소모 큼 |
| **심리적 부담** | "촬영 중"이라는 심리적 불편감 (본인 + 주변) |

### 2-2. 기구 부착형 횟수 센서

```
┌─────────────────────────────────────────────┐
│              피카 모션 센서                    │
│                                              │
│  ┌──────────────────────────────────────┐   │
│  │  [가속도 센서] + [자이로스코프]        │   │  ← 기구의 움직임 감지
│  │  → 반복 동작 패턴 인식                │   │     (위↔아래, 당김↔밀기)
│  │  → 횟수 자동 카운팅                   │   │
│  └──────────────────────────────────────┘   │
│                                              │
│  부착 위치: 바벨바, 케이블 손잡이, 덤벨 등    │
│  통신: BLE → Phase 1 컨트롤러 또는 앱        │
│  전원: 코인 배터리 (6개월+ 수명)             │
│  크기: 약 30×30×10mm (동전 크기)             │
└─────────────────────────────────────────────┘
```

### 2-3. Phase 2 구성

```
[기구 부착 모션 센서]  ──BLE──→  [Phase 1 스마트 컨트롤러]  ──BLE──→  [앱]
     │                                  │                            │
     │ 횟수 감지                        │ 무게 입력 (Phase 1)         │ 기록 저장
     │ (가속도+자이로)                  │ 세트/휴식 표시              │ 통계 분석
     │                                  │ 횟수 표시                  │ AI 추천
```

**장점:**
- 스마트폰 카메라 **불필요** (배터리 절약, 공간 해방)
- 기구에 직접 부착 → **각도/시야 문제 없음**
- 촬영 부담 없음 → **심리적 장벽 제거**
- 기존 센서 기반 IoT 접근과 달리, **기구 개조 불필요** (부착형)

## Phase 3: 자동 무게 감지 완성 (최종 목표)

**핵심 컨셉**: Aspect Ratio 기반 원판 장수 자동 감지 R&D 완성

### 3-1. 현재 달성 수준

| 항목 | 결과 |
|------|------|
| 원판 종류 식별 (YOLO) | **96.2%** (완료) |
| 1장 감지 | **100%** |
| 2장 감지 | **75%** (업계 최초) |
| 3장 감지 | 성공 사례 있음 (추가 검증 필요) |
| 4장 이상 | 미검증 (R&D 진행) |

### 3-3. R&D 로드맵

| 단계 | 목표 | 접근법 | 기간 |
|------|------|--------|------|
| **3-A** | 2장 감지 85%+ | Aspect Ratio + 서브픽셀 보정 + EMA 고도화 | 3개월 |
| **3-B** | 3장 감지 80%+ | 합성 데이터 학습 + 순서 회귀(Ordinal Regression) 모델 | 6개월 |
| **3-C** | 4장+ 감지 | Multi-cue 하이브리드 (DINOv2 + 기하학적 분석) | 12개월 |
| **3-D** | 완전 자동 무게 계산 | 원판 종류(YOLO) × 장수(Aspect Ratio) = 총 무게 자동 | 최종 |

**최종 목표:**
> 사용자가 바벨을 카메라 앞에 가져오면 → 원판 종류 + 장수를 자동 감지 → 총 무게 자동 계산
> → **무게 입력 0건** 달성 → Phase 1 하드웨어 입력 장치도 불필요해지는 완전 자동화

| 무게 감지 | ToF + 전용 원판 → **실패** | 전용 기구 (RFID) | 수동 입력 | **YOLO + Aspect Ratio** |
| 횟수 카운팅 | CV (ToF 보조) | 기구 내장 센서 | 수동 입력 | **CV + 하드웨어 센서** |
| 폼 체크 | 제한적 | 없음 | 없음 | **AI 실시간** (계획) |
| 초기 비용 | $2,495 (기구 포함) | 기구 교체 수백만원 | $0 (앱만) | **앱 무료 + 장치 합리적** |
| 확장성 | 홈짐 한정 | B2B 한정 | 소프트웨어만 | **B2C + B2B** |
