## 에이전트 1: CV 소프트웨어 대안 분석

### 7가지 대안 비교표

| 접근법 | 실현성 | 정확도 | V2log 적합? |
|--------|--------|--------|------------|
| **Multi-Class YOLO** (1장/2장/3장 별도 클래스) | ⭐⭐⭐⭐⭐ | 85-92% | ✅ 추천 |
| **Instance Segmentation** (YOLOv8-seg 마스크) | ⭐⭐⭐⭐ | 80-88% | 보조 |
| **OCR + YOLO 하이브리드** | ⭐⭐⭐⭐ | 75-85% (앞면만) | 보조 |
| **Direct Regression** (무게 직접 추론) | ⭐⭐⭐ | 70-82% | 비추 |
| **VLM API** (GPT-4V/Gemini Vision) | ⭐⭐ | 65-75% | ❌ 환각+비용 |
| **Sleeve 측정** (바벨 슬리브 길이 비율) | ⭐⭐ | 60-75% | ❌ Mode A와 동일 |
| **QR Code / RFID** | ⭐⭐⭐ | 98%+ | B2B만 가능 |

### 핵심 발견
1. **Multi-Class YOLO가 최선**: 기존 926장 데이터를 `plate_20kg_x1`, `plate_20kg_x2`, `plate_20kg_x3`으로 재라벨링하면 2-3주 내 구현 가능
2. **VLM(Gemini/GPT-4V)은 환각 문제**: 물체 카운팅에서 VLM이 자주 틀림 (학술 벤치마크에서 확인됨)
3. **OCR은 보조용**: 앞면 원판만 읽을 수 있음, 뒷면 원판 무게가 다르면 오답
4. **Direct Regression 비추**: 학습 안 한 무게 조합에서 보간 실패

### VLM 관련 경고 (플랜 접근법 1에 영향)
```
에이전트 평가:
- GPT-4V/Gemini Vision: 65-75% 정확도 (높은 분산)
- 환각 발생: 물체 카운팅에서 흔한 실패 패턴
- 레이턴시: 500-1500ms (로컬 TFLite 50-200ms 대비)
- 비용: 10,000 DAU 기준 연 $600k+

→ 우리 플랜의 VLM 검증 단계에서 이 점 확인 필요
→ 80%+ 못 넘으면 즉시 접근법 2(BLE)로 전환
```

## 에이전트 2: 하드웨어 솔루션 분석

### 무게 입력용 하드웨어

| 솔루션 | 비용 | 정확도 | 공간 해결 | 추천 |
|--------|------|--------|----------|------|
| **PUSH Band** (VBT 장비) | $289 + $1000/년 | 95-98% | ✅ | 비쌈 |
| **DIY ESP32 버튼** | $25-35 (약 3-5만원) | 100% (수동) | ✅ | ✅ 우리 플랜 |
| **스마트워치 앱** | $0 (기존 워치) | 100% (수동) | ✅ | Galaxy Watch 보유 |
| **로드셀 (바벨 장착)** | $65-85/바벨 | 99% | ✅ | B2B만 |
| **RFID 태그** | $50-100/바벨 + $1-2/원판 | 100% | ✅ | B2B만 |

### 횟수 카운팅용 하드웨어

| 솔루션 | 비용 | 정확도 | 상태 |
|--------|------|--------|------|
| **mmWave 레이더** | $50-100 | 95%+ | 학술 단계 |
| **초음파 센서** | $15 | 75-85% | 특정 운동만 |
| **천장 카메라** (B2B) | $500-1000/스테이션 | 95%+ | B2B만 |
| **폰 카메라 Pose** (현재) | $0 | 90%+ | ✅ 유지 |

### 핵심 발견
1. **PUSH Band는 너무 비쌈**: $289 + 연 $1000 구독 → 일반 사용자에게 추천 불가
2. **DIY ESP32가 현실적**: $25-35로 BLE 버튼 제작 가능, 우리 플랜의 접근법 2와 일치
3. **스마트워치 앱도 유력**: Galaxy Watch 보유 → 추가 비용 0원, 하지만 WearOS 별도 개발 필요
4. **폰 카메라 Pose 유지가 최선**: 횟수 카운팅은 현재 시스템이 가장 합리적

## 에이전트 3: 헬스장 공간 제약 + 경쟁사 분석 (⚠️ 전략적 중요)

### 경쟁사 현실

| 앱 | 방식 | 다운로드 | 핵심 |
|----|------|---------|------|
| **Fitbod** | 수동 입력 | 15M+ | AI 루틴 추천, 1RM 28% 빠른 성장 |
| **Strong/JEFIT** | 수동 입력 | 8M+ | 빠른 기록 (세트당 10-15초) |
| **Tempo Move** | 폰 카메라 CV | — | 홈짐 전용, 상업 헬스장 진출 실패 |
| **Kemtai/Onyx** | 폰 카메라 CV | — | 홈짐만, "sterile" 경험 피드백 |
| **VAY Sports** | 폰 카메라 CV | — | **서비스 종료 (deprecated)** |

### 상업 헬스장에서 폰 CV의 현실

| 항목 | 폰 CV (V2log) | 수동 입력 (Fitbod) |
|------|--------------|-------------------|
| **세트당 시간** | 2-3분 (설치+촬영) | 10-15초 (탭) |
| **피크 시간** | ❌ 공간 부족 | ✅ 문제 없음 |
| **다른 사람 통과** | CV 깨짐 | 영향 없음 |
| **프라이버시** | ⚠️ 다른 회원 촬영 우려 | 문제 없음 |
| **기구 이동 시** | 폰 재설치 필요 | 앱만 탭 |

### 핵심 발견 (⚠️ 냉정한 평가)

1. **Tempo Move도 상업 헬스장 실패**: 전용 하드웨어 + 폰 CV 조합으로도 못 뚫음
2. **수동 입력 앱이 시장 지배**: 세트당 10-15초의 편의성을 이길 수 없음
3. **폰 CV = 홈짐 전용**: 상업 헬스장에서는 설치 시간 + 공간 + 프라이버시 문제
4. **CV를 필수가 아닌 선택으로**: "자동 기록" → "폼 코칭 보조" 포지셔닝이 현실적

### 전략 제안 (에이전트 의견)

```
현재 V2log 포지셔닝: "전문가 루틴 + 빠른 기록"
문제: 폰 CV가 "빠른 기록"과 충돌 (설치 2-3분 > 수동 입력 15초)

제안된 포지셔닝:
- 기본 UX: Fitbod처럼 빠른 수동 입력 (10-15초)
- 부가 기능: CV는 "폼 코칭" or "무게 자동 제안"으로 optional
- 타겟: 홈짐 사용자 + 진지한 취미인 (상업 헬스장은 Phase 3+)
```

### Tempo가 발견한 것 (시장 신호)
- 전용 하드웨어(Tempo Studio) → 폰 전용(Tempo Move)로 피벗
- Tempo Move도 **홈짐 전용**으로만 작동
- 상업 헬스장 진출: **한 번도 성공 못 함**
- 결론: "잘 만든 폰 CV + 하드웨어 콤보도 상업 헬스장을 뚫지 못했다"

## 종합 판단: 우리 플랜과의 정합성

| 플랜 항목 | 에이전트 평가 | 리스크 |
|----------|-------------|--------|
| **접근법 1 (VLM Gemini)** | ⚠️ 에이전트 1이 65-75% 정확도 + 환각 경고 | **검증 필수** — 80% 못 넘을 가능성 있음 |
| **접근법 2 (ESP32 BLE)** | ✅ 에이전트 2가 $25-35 DIY 가능 확인 | 낮음 — 수동 입력이라 정확도 100% |
| **Pose 횟수 유지** | ✅ 에이전트 2, 3 모두 현재 방식이 최선 | 공간 제약만 해결 필요 |

### 추가 고려사항
- **Multi-Class YOLO** (에이전트 1 최우선 추천): VLM 실패 시 고려할 가치 있음
  - 기존 데이터 재라벨링으로 85-92% 가능 (에이전트 추정)
  - 다만 "겹친 원판이 시각적으로 다르게 보인다"는 가정이 실제 헬스장에서 성립하는지 미검증
- **Galaxy Watch 앱**: 접근법 2의 변형으로 고려 가능 (ESP32 대신 기존 워치 활용)
