# 헬스장 자동 운동 기록 시스템 — 독립 조사 프롬프트 (개방형)

> v2 개방형: Claude 조사 결론을 미리 노출하지 않고, AI가 스스로 결론을 도출하도록 설계
> v1(규정형)과 비교하여 어느 쪽이 더 유용한 결과를 내는지 확인용

---

## 배경설명

### 우리 회사
- **회사명**: 피카(Fica) — 피트니스 스타트업 (예비창업 단계, 한국)
- **대표**: 16년 피트니스 경력
- **목표**: 헬스장에서 회원의 운동 기록(횟수, 세트, 무게)을 자동으로 남겨주는 시스템 개발

### 시도한 것들과 결과
- **카메라 + AI(CV) 접근**: 스마트폰 카메라로 바벨 원판을 인식해 무게를 자동 감지하려 했으나 실패
  - 겹친 원판 인식 불가, VLM(대형 비전-언어 모델) 최고 71%로 목표 80% 미달
  - 횟수 카운팅은 MediaPipe BlazePose로 88~95% 달성 (성과)
- **결론**: 카메라로 무게 자동 감지는 포기. 다른 방법 모색 중

### 새로 검토 중인 방향
- 기구(머신, 바벨 등)에 **관성 측정 장치(IMU 센서)**를 부착하여 운동 횟수를 자동 카운팅
- BLE로 스마트폰 앱에 전송, NFC로 기구/사용자 식별
- 무게는 앱에서 수동 입력 (자동 감지 안 함)
- B2B 모델: 헬스장에 설치 → 회원 앱 사용

### 2025년 사업계획서 원안
- 센서: MPU6050 (6축 IMU)
- 장치: 12cm × 12cm × 2cm, BLE 5.0 + NFC
- 기구 프레임/바벨바/케이블 손잡이에 부착

## 프롬프트

```
당신은 피트니스 테크놀로지 분야의 독립 컨설턴트입니다.

위 배경의 스타트업이 "헬스장 기구에 IMU 센서를 부착하여 운동 횟수를 자동 카운팅하는 B2B 시스템"을 개발하려 합니다.

이 사업의 실현가능성을 처음부터 독립적으로 조사해주세요.
사전에 정해진 결론은 없습니다. GO일 수도, NO-GO일 수도 있습니다.

### 1. 시장 조사
- 이 분야에 이미 도전한 회사/제품이 있는가?
- 성공한 곳과 실패한 곳은? 실패 원인은?
- 학술 연구에서 비슷한 시도가 있었는가?
- 현재 시장에서 이 접근이 "블루오션"인가, 아니면 "이미 실패가 검증된 영역"인가?

### 2. 기술 실현가능성
- 사업계획서에 명시된 MPU6050 센서가 이 용도에 적합한가?
- 더 나은 센서 대안이 있다면 무엇인가?
- 헬스장의 다양한 운동 기구(머신, 바벨, 덤벨, 케이블 등)에서 각각 센서 기반 횟수 카운팅이 가능한가?
- 알고리즘은 어떤 수준의 정확도를 기대할 수 있는가?

### 3. 사업 리스크
- 이 사업이 실패할 수 있는 가장 큰 이유들은 무엇인가?
- 기술적 한계, UX 문제, 비즈니스 모델 문제를 모두 포함하여 분석
- "이것만 해결 못하면 사업 자체가 불가능한" 치명적 문제(쇼스토퍼)가 있는가?
- 기존 수동 입력 앱(Strong, Hevy 등)이나 스마트워치 대비 충분한 차별화가 되는가?

### 4. 종합 판정과 제안
- 위 조사를 종합하여 GO / CONDITIONAL GO / NO-GO 판정
- CONDITIONAL GO라면 어떤 조건이 충족되어야 하는가?
- 가장 현실적인 첫 진입 전략은? (어떤 시장, 어떤 기구, 어떤 고객부터)
- 센서 방식 외에 더 나은 대안이 있다면 자유롭게 제안
```

## v1(규정형)과 v2(개방형) 차이점

| 항목 | v1 (규정형) | v2 (개방형) |
|------|-----------|-----------|
| 쇼스토퍼 | 8개 미리 나열 → "확인" | 나열 안 함 → AI가 직접 "발견" |
| 센서 비교 | MPU6050/BMI270/ICM/LSM 4개 지정 | "더 나은 대안이 있는가?"만 질문 |
| 운동 목록 | 12종 개별 지정 | "다양한 기구에서 가능한가?"로 개방 |
| CV 실패 상세 | YOLO/MiDaS/Sobel/VLM 4가지 상세 | "CV로 시도했으나 실패" 한 줄 |
| 조사 영역 | 6개 세분화 | 4개 큰 영역 |
| 유도 방향 | BMI270, NFC 탭, 스마트워치 해법 암시 | 결론 없음 |

### 어떤 걸 먼저 쓸까?
- **v2(개방형) 먼저** → AI의 독립적 판단을 얻음
- **v1(규정형) 나중에** → Claude 결과를 공유하고 "이 분석에 동의하는가?" 토론
