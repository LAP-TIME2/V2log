### 아이템 개요

**"스마트폰 카메라가 당신의 트레이너가 된다"**

피카(Fica)는 스마트폰 카메라와 AI 컴퓨터 비전 기술만으로 **운동 횟수 자동 카운팅**, **실시간 자세(폼) 교정**, **AI 맞춤 피드백**을 제공하는 피트니스 코칭 플랫폼이다.

**핵심 기능:**
- **운동 자동 기록**: 카메라가 운동 종류, 반복 횟수, 세트를 자동 인식·기록 → 수동 입력 불필요
- **실시간 폼 체크**: AI가 33개 관절 각도를 분석하여 잘못된 자세를 즉시 교정 ("팔꿈치를 더 내리세요")
- **AI 맞춤 피드백**: 운동 이력 기반 최적 루틴·강도·휴식 시간 추천
- **무게 자동 인식** (단계적): Phase 1 수동 입력 → Phase 2 OCR/플레이트 인식 → Phase 3 완전 자동화

**기존 센서 방식 대비 핵심 전환점:**

| 항목 | 기존 (센서 디바이스) | CV 피벗 후 (스마트폰 카메라) |
|------|---------------------|---------------------------|
| 하드웨어 비용 | 기구당 5~10만원 × 50~80대 | **0원** (스마트폰 활용) |
| 자세 교정 | **불가능** | **실시간 가능** (킬러 기능) |
| 확장 속도 | 기구 설치 필요 → 느림 | 앱 다운로드 → **즉시 글로벌** |
| 양산/물류 | 금형·양산·재고 관리 필요 | **완전 제거** |

> Fica는 단순한 운동 기록 앱이 아니라, **"월 9,900원으로 PT 트레이너를 대체하는 AI 코칭 플랫폼"**이다.

## 1-1. 창업아이템 배경 및 필요성

### (1) 급성장하는 피트니스 시장과 해결되지 않은 구조적 문제

글로벌 피트니스 시장은 2025년 약 **186억 달러(약 25조 원)** 규모이며, 2035년까지 **598억 달러**로 성장 전망 (CAGR 12.3%, Future Market Insights). AI 피트니스 시장은 더욱 빠르게 성장하여 2024년 **98억 달러** → 2034년 **461억 달러** (CAGR 16.8%, InsightAce Analytic). 국내 피트니스 시장 역시 2025년 약 **47.5억 달러(약 6.3조 원)**, CAGR 9.96%로 안정적 성장세를 보인다 (Mordor Intelligence).

그러나 이 거대한 시장의 이면에는 해결되지 않은 핵심 문제들이 존재한다.

**[문제 1] 운동 데이터의 수동 기록 — "기억에 의존하는 운동"**

헬스장에서 웨이트 트레이닝을 수행하는 대다수의 이용자는 운동 기록을 체계적으로 관리하지 못한다. 세트마다 폰을 꺼내 숫자를 입력하는 것은 운동 흐름을 깨뜨리며, 대부분 2주 만에 포기한다. "지난주에 벤치프레스를 몇 kg으로 했는지" 기억하지 못한 채 매번 감(感)에 의존하여 운동을 반복하는 것이 현실이다.

이는 **과학적 점진적 과부하(Progressive Overload)의 실패**로 직결된다. 기록 없이는 중량이나 횟수를 점진적으로 늘려나갈 근거가 없으며, 정체기 → 회의감 → 운동 중단의 악순환을 유발한다.

**[문제 2] 자세 피드백의 부재 — "혼자서는 알 수 없는 폼"**

올바른 자세(폼)는 운동 효과 극대화와 부상 예방의 가장 중요한 요소이다. 그러나 거울만으로는 측면 각도, 관절 가동 범위, 좌우 대칭성을 정확히 확인하기 어렵다. 이를 해결하는 유일한 방법은 퍼스널 트레이닝(PT)인데, 서울 기준 **1회 5~7만 원, 월 12회 기준 60~84만 원**으로 대다수에게 부담이 크다.

**[문제 3] 헬스장 기구 혼잡도 관리의 부재**

피크 시간대(오후 6시~9시) 인기 기구(벤치프레스, 스쿼트랙 등) 앞 평균 대기 인원 2.5명. 운동 시간의 30~40%가 대기에 소비되지만, 실시간 기구 사용 현황을 파악하는 기술적 인프라가 전무하다.

**[문제 4] 운동 지속성의 위기 — "3개월의 벽"**

헬스장 신규 회원의 약 50%가 6개월 이내 이탈, 연간 회원 유지율은 50~60%에 불과. 핵심 원인은 **눈에 보이는 성과의 부재**와 **개인 맞춤 피드백의 부족**이다.

### (2) 기존 센서 방식의 구조적 한계

피카(Fica)는 창업 초기, 12cm×12cm 크기의 IoT 센서 디바이스(Hall센서+NFC+가속도센서)로 문제 해결을 시도하였다. 그러나 프로토타입 개발 과정에서 다음의 **구조적 한계**가 드러났다.

| 한계 | 상세 |
|------|------|
| **하드웨어 비용** | 기구당 BOM+조립비 발생, 50~80대 기구에 전면 도입 시 수백만 원 |
| **기구별 호환성** | 제조사/모델마다 구조가 상이 → 기구마다 별도 마운팅·캘리브레이션 필요 |
| **양산 복잡성** | 소량 다품종 제조·물류·AS의 전 과정이 린(Lean) 운영과 충돌 |
| **설치 저항** | 운영자의 기구 보증 문제, 미관, 관리 부담에 대한 심리적 저항 |
| **기능적 한계** | 센서는 중량·움직임만 감지 가능, **자세(폼) 분석은 원천적으로 불가능** |
| **확장 속도** | 1곳 도입에 현장 설치+교육 필요 → 소프트웨어의 "다운로드 한 번"에 비해 수십 배 느림 |

> **핵심 교훈**: 센서로는 절대 자세 교정(폼체크)을 할 수 없다. 소프트웨어만으로 해결할 수 있는 문제에 하드웨어를 끼워 넣는 것은 시장 확장의 병목이 된다.

### (3) 기술 환경의 전환점: 컴퓨터 비전의 성숙

2025~2026년을 기점으로, 스마트폰 기반 Computer Vision 기술이 피트니스 영역에 본격 적용 가능한 수준에 도달하였다.

**Pose Estimation 정확도**: MediaPipe Pose 기반 관절 각도 예측 **97.68% 정확도** 달성 (2025 FusionPose 논문, ScienceDirect)

**실시간 성능**: CPU만으로 **30+ FPS** 달성, GPU 불필요. YOLO11-n 객체 탐지 17ms (~57 FPS)

**온디바이스 AI**: 최신 스마트폰 NPU가 CPU 대비 5~10배 가속. 서버 전송 없이 개인정보 보호하며 실시간 처리 가능

**학술 검증**: 카메라 기반 반복 횟수 카운팅 정확도 **90~97%** (Fitcam 2024, Nature Scientific Reports 2025), 운동 분류 정확도 **97~99%** (BiLSTM 연구, arXiv)

> **이 세 가지 기술 성숙 요인의 결합은, 수억 원 규모의 하드웨어 투자 없이도 "운동 자동 기록 + 실시간 자세 교정"을 스마트폰 앱 하나로 실현할 수 있는 전환점을 만들었다.**

### (4) 페르소나 시나리오: "이OO씨의 변화"

> **이OO씨 (42세, IT기업 개발자, 운동 경력 2년)**
>
> **Before — 피카 없이:**
> 퇴근 후 헬스장, 오후 7시. 벤치프레스 앞에 두 명이 대기 중이다. 10분 기다려 자리를 잡았다. "지난주에 70kg 8회였나, 75kg 6회였나..." 기억이 안 난다. 기록 앱을 써봤지만 세트마다 폰 입력이 귀찮아 2주 만에 포기했다. 감으로 70kg을 잡는다. 세 번째 세트, 어깨에 통증. 폼이 잘못된 건지, 중량이 과한 건지 모른다. PT를 받고 싶지만 월 84만 원은 부담이다. 3개월째 변화 없다. 또 헬스장을 끊을까 고민한다.
>
> **After — 피카와 함께:**
> 스마트폰을 거치대에 세우고 피카 앱을 켠다. "지난 세션: 벤치프레스 72.5kg × 8회 × 4세트, **추천: 75kg × 7회**" 데이터 기반 가이드가 뜬다. 운동 시작. 카메라가 자동으로 동작을 인식하고 횟수가 실시간 카운트된다. 세 번째 세트에서 **"왼쪽 팔꿈치가 과도하게 벌어지고 있습니다. 견갑골을 더 조여주세요"** 음성 피드백. 운동 종료 후 총 볼륨, 1RM 변화 추이가 자동 정리된다. **"4주간 1RM 82kg → 87kg 향상"** — PT 없이도, 데이터가 "당신은 제대로 하고 있다"고 말해준다.

## 2-1. 창업아이템 개발/개선 준비현황

### (1) 기존 V2log 앱 개발 완료 — "이미 작동하는 제품"

피카의 모체 서비스 **V2log**는 Flutter 기반 크로스플랫폼 앱으로 **MVP 개발이 완료**된 상태이다. CV 피벗 시 앱 프론트엔드와 백엔드 인프라를 그대로 활용하여 **약 4~6개월의 개발 기간 단축** 효과가 있다.

#### V2log 기(旣)구현 기능

| 영역 | 구현 완료 기능 | 기술 스택 |
|------|--------------|----------|
| 인증/온보딩 | 이메일 회원가입/로그인, 프로필, 온보딩 | Supabase Auth |
| 운동 기록 | 세트별 무게/횟수, 빠른 입력 UI(10초 기록), 세트 타입(웜업/본세트/드롭/슈퍼세트) | Flutter + Riverpod |
| 루틴 관리 | 전문가 프리셋 루틴 5종, 루틴 라이브러리 | Supabase DB |
| 통계 분석 | 주간/월간 볼륨 차트, 부위별 분석, 1RM 진행도, 강도 존 분석 | fl_chart |
| 운동 라이브러리 | 운동 검색/필터, 시범 이미지, 상세 가이드, 근육 맵 시각화 | muscle_selector |
| 커뮤니티 | 오운완 공유 카드 위젯, 이미지 저장/공유 | share_plus, gal |
| 부가 기능 | 휴식 타이머, 알림 설정(요일별), 다크/라이트 테마 | flutter_local_notifications |

## 2-2. 창업아이템 실현 및 구체화 방안

### (1) 핵심 기술 구조: 센서 → CV 전환

| 측정 항목 | 기존 (센서) | CV 피벗 후 | 장점 |
|----------|-----------|-----------|------|

| 무게 (핀머신) | Hall 센서 | **OCR 카메라 (숫자판 인식)** | 기구 개조 불필요 |
| 무게 (플레이트) | RFID 태그 | **YOLO 객체 인식** | 태그 부착 불필요 |
| 기구 인식 | NFC 태그 | **카메라 위치 학습 / QR** | 인프라 변경 불필요 |
| **폼 체크** | **불가능** ❌ | **관절 각도 임계값 비교** ✅ | **신규 킬러 기능!** |
| 트래픽 관리 | 디바이스 감지 | **카메라 기구 사용 현황** | 별도 인프라 불필요 |

### (2) 작동 시나리오

```
사용자 행동                    시스템 처리                     화면 출력
────────────────────────────────────────────────────────────────────
1. 스마트폰 거치대에         앱 실행, 카메라 활성화
   놓고 촬영 시작

2. 운동 시작 자세 취함  →   MediaPipe가 33개 관절 추출       "벤치프레스
                           운동 분류 모델 자동 인식            감지했습니다"

3. 첫 번째 반복 수행   →   관절 각도 변화 추적              횟수 카운터: 1
                           Peak→Valley→Peak = 1회

4. 폼 흐트러짐         →   팔꿈치 각도 기준값 벗어남         "팔꿈치를 더
                           임계값 비교 → 오류 감지             내리세요" 🔊

5. 세트 완료 (10회)    →   3초간 미동작 → 세트 종료         "10회 완료!
                           자동 기록 저장                     60kg"

6. 전체 운동 종료      →   운동 요약 자동 생성               총 볼륨, 1RM,
                           기존 통계 시스템에 반영             부위별 분석
```

### (4) 폼 체크 알고리즘 — CV 피벗의 핵심 차별점

| 운동 | 체크 관절 | 정상 범위 | 폼 오류 판정 | 피드백 |
|------|----------|----------|-------------|--------|
| 벤치프레스 | 팔꿈치 | 80~100도 | <70도 또는 >120도 | "팔꿈치 각도를 조절하세요" |
| 스쿼트 | 무릎 | 70~90도 | 무릎이 발끝 과도 초과 | "무릎이 발끝을 넘지 않게" |
| 데드리프트 | 허리(척추) | 170~180도 | <160도 (라운딩) | "허리를 펴세요! 부상 위험" |
| 오버헤드프레스 | 어깨+팔꿈치 | 90~180도 | 허리 과신전 | "코어에 힘을 주세요" |

> **관절 각도 기준값 DB**는 대표자의 16년 트레이닝 경력과 CES/CMS 전문 자격에 기반하여 구축된다. 이는 논문에서 가져온 데이터와는 차원이 다른 **현장 검증된 기준값**으로, 피카의 핵심 경쟁 우위이자 데이터 해자(Data Moat)이다.

## 3-2. 시장 진입 전략

### 기본 방향: "미국 파일럿 → 글로벌 동시 출시 → 한국 확대"

CV 피벗의 가장 큰 전략적 이점은 **소프트웨어이므로 글로벌 동시 출시 가능**하다는 것이다.

```
[ 기존: 센서 기반 해외 진출 ]
기획 → 설계 → 금형 → 양산 → FCC인증 → 물류 → 판매 → AS
                ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                12~18개월 + 수억원

[ CV 피벗: 앱 기반 해외 진출 ]
기획 → 개발 → 앱스토어 출시 → 마케팅
              ^^^^^^^^^^^^^^^^
              3~6개월 + 인건비 중심
```

**V2log 기존 자산 활용:**
- 운동 기록/통계 시스템 → CV 자동 기록의 저장·분석 인프라로 즉시 활용
- 프리셋 루틴 라이브러리 → AI 추천의 기초 데이터베이스
- 1RM/PR 추적 → CV 자동 기록과 연동하여 무입력 자동 갱신
- 오운완 공유 카드 → CV 분석 결과 포함 프리미엄 카드로 확장

## 팀의 경쟁 우위

```
┌─────────────────────────────────────────────────────┐
│                    피카(Fica) 팀                      │
│                                                     │
│   도메인 전문성        ×        기술 실행력            │
│   ─────────────              ─────────────          │
│   · 16년 트레이너              · 삼성전자 AI 출신     │
│   · CES/CMS 자격              · V2log 앱 개발 완료   │
│   · 수천 명 자세 교정           · Flutter + Supabase  │
│   · 셀럽 트레이닝 이력          · CV/ML 전환 역량      │
│                                                     │
│                    ↓                                 │
│       "정확한 기준값" + "이를 구현하는 기술"             │
│       = 경쟁사가 쉽게 복제할 수 없는 진입 장벽          │
└─────────────────────────────────────────────────────┘
```

> 피트니스 AI 시장에서 "기술만 있는 팀"은 정확한 운동 기준을 모르고, "트레이너만 있는 팀"은 이를 구현할 수 없다. **도메인 전문성과 기술 실행력을 동시에 보유한 팀 구성**이 피카의 가장 근본적인 경쟁 우위이다.
