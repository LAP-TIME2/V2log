| 항목 | 사양 |
|------|------|

### W8-Scope (Microsoft Research) — 가장 유사한 연구
- **웨이트 스택 상단에 가속도계 + 자력계 부착** ← 피카와 가장 비슷한 접근
- 랫풀다운 96%, OHP 97%, 벤치프레스 98% 정확도 달성
- 자력계로 핀 위치(무게 설정)까지 자동 감지
- **단, 연구 수준이며 상용화되지 않음**

### 핵심 인사이트
> **"기구 프레임에 IMU를 부착하는 상용 제품"은 아직 존재하지 않음 → 블루오션**
> 기존 제품들은 모두 "손목/팔 착용" 또는 "바벨 클램프" 방식

## 3. 핵심 발견 — 기술적 실현가능성

### MPU6050은 쓰면 안 된다

| 항목 | MPU6050 (사업계획서) | **BMI270 (권장)** |
|------|---------------------|-------------------|
| 노이즈 | 높음 (5,000 mdps/√Hz급) | **극저 (7 mdps/√Hz)** |
| 드리프트 | 수 분 내 정확도 급감 | 매우 안정적 |
| 온칩 ML | 없음 | **내장 (동작 분류 가능)** |
| 가격 | $0.5~1 | $1~2 |
| 사용처 | Arduino 학습용 | Apple Watch, Garmin 등 |

> **결론: BMI270으로 교체 필수. 가격 차이 거의 없고 성능은 수십 배 차이**

### 운동 종류별 실현가능성 매트릭스

| 운동 | 판정 | 센서 부착 위치 | 근거 |
|------|------|------------|------|
| 벤치프레스 (바벨) | **◎ 매우 가능** | 바벨 중앙/끝 | 수직 왕복, 논문 다수 검증 |
| 스쿼트 (바벨) | **◎ 매우 가능** | 바벨 끝 | 수직 변위 크고 규칙적 |
| OHP (바벨) | **◎ 매우 가능** | 바벨 중앙 | 수직 가속도 명확 |
| 레그프레스 | **◎ 매우 가능** | 무게 캐리지 | 고정 레일, 가장 쉬운 조건 |
| 스미스 머신 | **◎ 매우 가능** | 바 or 레일 | 수직 단축 고정 |
| 체스트프레스 머신 | **◎ 매우 가능** | 웨이트 스택/암 | 수평 왕복 명확 |

## 4. 핵심 발견 — 비즈니스/UX 비판

### 쇼스토퍼 3개 발견

| # | 문제 | 심각도 | 설명 |
|---|------|--------|------|
| **1** | **다중 사용자 구분** | **치명적** | 같은 기구를 여러 명이 쓸 때 "내 기록"을 분리하는 방법이 없음. NFC 탭 강제 → 마찰, 안 하면 → 데이터 오염 |
| **2** | **덤벨/자유중량 미지원** | **치명적** | 헬스장 운동의 40~50%가 덤벨. 이걸 지원 못하면 "절반만 작동하는 제품" |
| **3** | **기구별 호환성** | **상** | 기구 100종 × 부착 위치 × 보정값 = 관리 케이스 기하급수 |

### 추가 위험 요소

| 문제 | 심각도 | 설명 |
|------|--------|------|
| 기구 보증 무효화 | 상 | 제3자 장치 부착 시 Technogym/Life Fitness 보증 무효 |
| 배터리/충전 관리 | 중~상 | 50대 장치 → 누가 충전? |
| 헬스장 도입 비용 | 중 | 50대 × 10만원 = 500만원 초기 투자 |
| 스마트워치 경쟁 | 중 | Apple Watch/Galaxy Watch가 이미 일부 횟수 추적 |
| 기존 앱과 전환 동기 | 중 | Strong/Hevy 수동 입력이 "충분히 빠름" |
| 헬스장 폐업률 | 중 | 5년 내 81.6% 폐업 → B2B 고객 유지 어려움 |

### Beast Sensor 실패가 주는 교훈
> "어떤 운동에서는 모든 횟수를 잡았고, 다른 운동에서는 하나도 못 잡았다. 고스트 랩(유령 횟수)이 계속 나왔다."
> → **90% 정확도여도 나머지 10%의 오류 때문에 사용자는 결국 직접 세게 됨**

## 5. 종합 판정

### 결론: 기술적으로는 가능하나, UX/비즈니스 구조가 핵심

| 항목 | 판정 | 설명 |
|------|------|------|
| 센서로 횟수 카운팅 | **✅ 가능** | 머신 계열 85~98%, 바벨 90~97% |
| MPU6050 사용 | **❌ 교체 필수** | BMI270으로 변경 (같은 가격, 수십배 성능) |
| 머신 운동 지원 | **✅ 가능** | 핀로드/플레이트로드 머신 = 가장 유리 |
| 바벨 운동 지원 | **✅ 가능** | 바벨 클램프 형태로 부착 |
| 덤벨 운동 지원 | **❌ 불가** | 모든 덤벨에 센서 부착 비현실적 |
| 다중 사용자 구분 | **⚠️ 해결 필요** | NFC 탭 or BLE 자동 연결 설계 필수 |
| 기구 범용 호환 | **⚠️ 해결 필요** | 기구별 프로파일 DB 구축 필요 |

### 가장 현실적인 첫 진입 전략 (에이전트 3팀 공통 의견)
> **일반 헬스장이 아니라 PT 스튜디오(회원 5~20명, 기구 10~20대)에서 시작**
> - 트레이너가 장치 관리
> - 회원 수 적어 다중 사용자 문제 완화
> - 기구 종류 통제 가능

## 7. 결정 반영 — 쇼스토퍼 해결 방안

### 쇼스토퍼 #1 해결: 다중 사용자 구분

일반 헬스장에서 여러 명이 같은 기구를 쓸 때 → **NFC 태깅을 의무화**

**현실적 UX 흐름:**
```
1. 회원이 기구에 도착
2. 스마트폰(또는 스마트워치)을 기구의 NFC 태그에 탭 (1초)
3. → 앱이 "벤치프레스 시작" 자동 표시 + 이전 기록 팝업
4. 운동 시작 → 기구 센서가 횟수 카운팅
5. → BLE로 해당 회원 앱에만 전송
6. 세트 완료 → 자동 휴식 타이머 시작
7. 다른 사람이 끼어들어 쓰면? → 그 사람도 NFC 탭 필요 (안 하면 기록 안 됨)
```

**왜 NFC 탭이 "마찰"이 아닌가:**
- 헬스장 입장할 때 이미 카드/앱 태깅을 함 → 익숙한 행동
- "탭 한 번"의 보상이 명확: 이전 기록 팝업 + 무게 추천 + 자동 카운팅
- 트래픽 관리 기능과 연결: 탭 = "이 기구를 지금 내가 쓰고 있다" 등록

### 쇼스토퍼 #2 해결: 덤벨 → 스마트워치

```
[머신/바벨]              [덤벨/자유중량]
     │                        │
  기구 부착 센서            스마트워치 IMU
  (BMI270)                 (Apple Watch/Galaxy Watch)
     │                        │
     └───── BLE ──→ 앱 ←── BLE ───┘
                    │
               통합 기록
```

**스마트워치 횟수 카운팅 현황:**
- Apple Watch: watchOS의 HealthKit + CoreMotion API로 운동 횟수 접근 가능
- Galaxy Watch: Samsung Health SDK + Wear OS 센서 API
- 학술 논문 기준 손목 IMU 정확도: 70~85% (기구 센서보다 낮지만 "보조 수단"으로 충분)

**장점:**
- 스마트워치 보급률이 이미 높음 (추가 하드웨어 비용 없음)
- 덤벨뿐 아니라 풀업, 딥스 같은 자유중량도 커버

**단점:**
- 스마트워치 미보유 회원은 수동 입력 (대안 필수)
- 정확도가 기구 센서보다 낮음 → "보조"로 포지셔닝

### 쇼스토퍼 #3 해결: 기구별 호환성 → 프로파일 시스템

```
[기구 프로파일 DB]
│
├── 랫풀다운계열 → 센서 위치: 스택 상단, 축: Z, 필터: 5Hz LP
├── 레그프레스계열 → 센서 위치: 캐리지, 축: 45°, 필터: 8Hz LP
├── 체스트프레스계열 → 센서 위치: 암, 축: X, 필터: 6Hz LP
├── 바벨계열 → 센서 위치: 바벨 중앙, 축: Z, 필터: 10Hz LP
└── ...
```

**작동 방식:**
1. 헬스장 도입 시 → 기구마다 NFC 태그에 "기구 종류 ID" 기록
2. 회원이 NFC 탭 → 앱이 기구 종류 인식 → 해당 프로파일의 필터/임계값 적용
3. 센서는 동일한 하드웨어, 소프트웨어 프로파일만 다름
4. 새 기구 추가 시 → 앱 업데이트로 프로파일 추가 (펌웨어 업데이트 불필요)

**현실성:**
- 기구 종류를 "운동 패턴"별로 그룹화하면 10~15개 프로파일이면 커버 가능
- 수직 왕복(스쿼트류), 수평 왕복(프레스류), 경사 왕복(레그프레스류), 케이블(풀다운류)
- 기구 제조사와 무관하게 "움직임 패턴"이 같으면 같은 프로파일 사용

### 소프트웨어 구성

```
[피카 센서] ──BLE──→ [V2log 앱] ←──BLE── [스마트워치]
     │                   │                    │
  머신/바벨            통합 기록              덤벨/자유중량
  횟수 카운팅         + 트래픽 관리           횟수 카운팅
  (85~98%)           + AI 추천               (70~85%)
                     + 운동 가이드
                     + 무게 수동 입력 (빠른 UX)
```

|---------|----------|-----------|
| 다중 사용자 구분 | NFC 탭 의무화 + 트래픽 연동 | 탭 안 하는 사용자 → 기록 안 됨 (수동 입력 대안) |
| 덤벨 미지원 | 스마트워치 연동 | 워치 미보유 시 수동 입력 |
| 기구 호환성 | 운동 패턴별 프로파일 DB (10~15종) | 특수 기구는 추가 프로파일 필요 |
| 기구 보증 무효 | 자석 탈부착 (비파괴) | 일부 브랜드 정책 확인 필요 |
| 배터리 관리 | USB-C 충전 + 앱 잔량 모니터링 | 헬스장 직원 교육 필요 |
