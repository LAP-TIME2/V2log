# /devlog — 개발 일지 업데이트

## 설명
DEVLOG.md에 빠진 커밋을 catch-up하고, `(auto)` 항목에 배운 점/결정 맥락을 보강합니다.

## 실행 조건
- 사용자가 `/devlog` 입력 시 실행
- 인자 없음

## 동작 순서

### 1. 빠진 커밋 확인
```bash
git log --format="%ai|%h|%s" --reverse
```
- DEVLOG.md에서 마지막 기록된 커밋 해시 확인
- git log에서 DEVLOG.md에 없는 커밋 찾기 (해시 비교)

### 2. 빠진 커밋 추가
- 날짜별로 그룹핑
- 해당 날짜 섹션에 커밋 추가 (없으면 새 섹션 생성)
- 형식:
  ```markdown
  ## YYYY-MM-DD (요일) — 제목

  #### 커밋
  - `커밋 메시지` — 해시
  ```

### 3. (auto) 항목 보강
- `#### 커밋 (auto)` 섹션 찾기
- 각 커밋에 대해:
  - CLAUDE.md Phase 매핑 확인
  - 관련 문서 링크 추가 (CV_수정노트, reference 보고서)
  - 가능하면 "배운 점" / "핵심 결정" 추가
- `(auto)` 태그를 `(보강됨)` 으로 변경

### 4. Part 1 경영진 요약 갱신
- Phase 요약 테이블: 날짜/상태/성과 업데이트
- 핵심 숫자: 총 커밋 수, 운동 종목 수 등 갱신
- 주요 마일스톤: 새 마일스톤 추가

### 5. 결과 출력
```
✅ DEVLOG.md 업데이트 완료
- 추가된 커밋: N개
- 보강된 (auto) 항목: N개
- Phase 요약 갱신: ✅
```

## 주의사항
- DEVLOG.md 파일이 `docs/DEVLOG.md`에 없으면 "DEVLOG.md가 없습니다. 먼저 생성해주세요." 출력
- 이미 있는 커밋은 중복 추가하지 않음 (해시로 확인)
- 기존 수동 작성 내용은 절대 덮어쓰지 않음
- `(auto)` 태그가 있는 항목만 보강 대상
